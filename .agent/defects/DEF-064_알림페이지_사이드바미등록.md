| **관련 Stage** | 전체 — 오더 상태 변경 알림, 배정 알림 등 |

## 현상

`/notifications` 페이지(`getNotifications` API 연동, 읽음 처리 포함)가 구현되어 있으나
NaviSidebar에 등록되지 않아 URL 직접 입력 없이 접근 불가.

## 기능 설명

- 사용자별 알림 목록 조회 (`getNotifications(50, 0)`)
- 읽지 않은 알림 수(`unreadCount`) 표시
- 알림 읽음 처리

## 권장 조치

사이드바 하단 또는 헤더 영역 Bell 아이콘에 `/notifications` 링크 연결.
별도 사이드바 항목보다는 **헤더 알림 버튼 → `/notifications` 라우팅** 방식 권장.

## 관련 파일

| 파일 | 설명 |
|:---|:---|
| `src/app/[locale]/(dashboard)/notifications/page.tsx` | 알림 센터 페이지 |
| `src/components/layout/NaviSidebar.tsx` | 또는 헤더 컴포넌트 연결 |
