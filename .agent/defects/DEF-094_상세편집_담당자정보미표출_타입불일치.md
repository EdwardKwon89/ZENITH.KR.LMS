## 발견 경위

UAT 화주 상세편집(`/agency/shippers/{id}/edit`) 화면 확인 중 발견.  
TASK-B-047(DEF-091)에서 `zen_organizations`에 contact 컬럼 3종이 추가된 이후 타입 재생성이 누락됨.

## 현상

**Defect A: Supabase TypeScript 타입 미재생성**
- `src/types/supabase.ts`의 `zen_organizations.Row`에 `contact_name`, `contact_email`, `contact_phone` 컬럼 없음
- 원인: migration `20260704101500_agency_003_org_contact_columns.sql` 적용 후 `npx supabase gen types typescript` 미실행
- 영향: `_createShipperOrg` insert, `getAgencyShipperById` select, `updateAgencyShipper` update에서 TypeScript 타입 오류 발생 → `next build` 실패 가능

**Defect B: 담당자 정보 미표출**
- 화주 상세편집 화면에서 담당자명·이메일·연락처 항목이 빈 값으로 표출
- 원인: 로컬 DB 쿼리 결과, 기존 테스트 화주(`"UAT agency shipper"`)의 contact 컬럼 값이 모두 NULL
  - 해당 화주는 contact 정보 입력 없이 생성됨 (선택 입력)
  - Defect A(타입 오류)로 인해 새 화주 등록 시에도 contact 데이터 저장이 불완전할 가능성 있음

## 영향 범위

- `src/types/supabase.ts` — `zen_organizations` Row/Insert/Update 타입 (contact 컬럼 누락)
- `src/app/actions/agency/shippers.ts` — `_createShipperOrg`, `getAgencyShipperById`, `updateAgencyShipper`
- 프로덕션 빌드(`next build`) 타입 체크 실패 위험

## DB 확인 결과

```
zen_organizations 컬럼: ... contact_name (TEXT), contact_email (TEXT), contact_phone (TEXT) — 존재 확인
현재 테스트 화주 contact 데이터: 모두 NULL
```

## 권장 조치

| 조치 | 명령 |
|:-----|:-----|
| Supabase 타입 재생성 | `npx supabase gen types typescript --local > src/types/supabase.ts` |
| TypeScript 오류 확인 | `rtk npm run build 2>&1 \| grep contact` |
| 엔드투엔드 검증 | 새 화주를 contact 정보 포함하여 등록 → 상세편집 화면에서 데이터 표출 확인 |
