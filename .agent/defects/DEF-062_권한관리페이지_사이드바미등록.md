# DEF-062: 권한 관리 페이지 사이드바 미등록

| 항목 | 내용 |
|:---|:---|
| **DEF#** | DEF-062 |
| **제목** | `/admin/permissions` (역할별 권한 관리) 페이지가 NaviSidebar에 미등록 |
| **관련 Stage** | STAGE 0 — 기초정보 설정 / 시스템 설정 |
| **유형** | UX 결함 / 접근성 누락 |
| **블로킹** | Y — ADMIN이 역할별 메뉴 접근 권한을 설정할 수 없음 |

## 현상

`/admin/permissions` 페이지가 구현되어 있으나 NaviSidebar에 등록되지 않아
ADMIN도 URL 직접 입력 없이는 접근 불가.

## 기능 설명

- 역할(ADMIN / MANAGER / CARRIER 등)별로 허용 경로(`ALL_RESOURCE_PATHS`) 권한 설정
- `rbac` 시스템과 연동 — 권한 변경 시 사이드바 노출 항목이 실시간 제어됨
- ADMIN 전용 (`requireAuth` + role 체크)

## 권장 조치

`NaviSidebar.tsx` `admin/settings` 하위 또는 `settings` 단독 항목으로 추가:

```ts
{ title: t("permissions"), href: "/admin/permissions" }
```
