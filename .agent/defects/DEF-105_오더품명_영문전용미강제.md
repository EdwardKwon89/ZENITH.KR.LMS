# DEF-105: 오더 품명(item_name) 입력란에 언어 제약이 없어 한글 입력이 SHXK 신고 영문 품명으로 그대로 전송됨

## 발견 경위
JSJung이 `SHXK_TEST_MOCK=false`로 오더 ZEN-2026-000001 실제 createorder 테스트 중 발견(DEF-103 다음, IMP-142/unit_code보다 먼저 발생):
```
申报英文品名不能包含除字母、数字和常用符号之外的特殊字符【청바지】
```
→ "신고 영문 품명에 문자·숫자·일반 기호 외의 특수문자(한글)를 포함할 수 없음"

## 근본 원인
`OrderRegistrationForm.tsx`의 품명(`item_name`) 입력란(`register(...items.${k}.item_name)`, 96~105행)에 언어 제약이 전혀 없음. `src/lib/validation/order.ts:8`의 zod 스키마도 `z.string().min(1, ...)`뿐이라 어떤 문자든 통과함.

`zen_order_items.item_name`이 `buildInvoiceFromItems()`(`src/lib/ups/label-mapping.ts:29`)에서 SHXK `invoice[].invoice_enname`(영문 품명, 필수)에 그대로 매핑되는데, 이 오더의 품목 4건 전부 한글로 등록돼 있었음(청바지/티셔츠/팜플렛/섬유원단). Mock 모드에서는 이 검증이 없어 지금까지 발견 안 됨(DEF-103·DEF-104와 동일 패턴).

## 영향 범위
품명을 한글로 입력한 모든 오더의 실제 SHXK createorder — mock 모드에서는 은폐됨.

## 긴급도
Medium — 이번 발견 건은 Jaison이 DB에서 직접 영문으로 정정 완료(청바지→Jeans 등)해 즉시 영향은 해소됨. 다만 원천 차단 장치가 없어 신규 오더에서 계속 재발 가능.

## 권장 조치
`orderItemSchema`(`src/lib/validation/order.ts`)의 `item_name`에 영문/숫자/일반기호 전용 정규식 검증 추가 + 폼에 에러 메시지 표시. 상세 설계는 Issue #574 참조.
