# DEF-061: 마스터 포장 메뉴 사이드바 미등록

| 항목 | 내용 |
|:---|:---|
| **DEF#** | DEF-061 |
| **제목** | `/master-orders` (마스터 포장) 페이지가 NaviSidebar에 미등록 |
| **관련 Stage** | STAGE 5 — 마스터 포장·출고 |
| **유형** | UX 결함 / 접근성 누락 |
| **블로킹** | Y — 창고 운영자(OPERATOR)가 마스터 오더 생성 기능에 접근 불가 |
| **긴급도** | High |
| **상태** | 미수정 |
| **담당자** | 미배정 |

---

## 현상

`/master-orders` 페이지(마스터 오더 생성 및 Packing List 출력)가 구현되어 있으나,
`NaviSidebar`의 `NAV_ITEMS`에 등록되지 않아 사이드바 메뉴로 접근할 방법이 없음.

URL을 직접 입력(`/[locale]/master-orders`)해야만 접근 가능한 상태.

---

## 영향 범위

| 영역 | 내용 |
|:-----|:-----|
| 대상 역할 | 창고 운영자(OPERATOR), 플랫폼 운영자(MANAGER), ADMIN |
| 기능 누락 | 하우스 오더 → 마스터 오더 묶음 생성 (WAREHOUSED → PACKED) |
| 기능 누락 | Packing List PDF 출력 (`/master-orders/{id}/packing`) |
| 정상 동작 | 출고 처리(`/warehouse/outbound`, PACKED → RELEASED)는 사이드바 정상 등록됨 |

---

## 원인

`src/components/layout/NaviSidebar.tsx`의 `NAV_ITEMS` 배열에 `/master-orders` 항목이 추가되지 않음.

현재 "물류 관리" 그룹 구성:
```
logistics_group (/tracking)
  ├── logistics_tracking  → /tracking
  ├── inventory           → /inventory
  ├── logistics_inbound   → /warehouse/inbound
  └── logistics_outbound  → /warehouse/outbound   ← /master-orders 누락
```

---

## 권장 조치

`NaviSidebar.tsx` `logistics_group` 하위에 아래 항목 추가:

```ts
{ title: t("master_orders"), href: "/master-orders" }
```

`messages/ko.json` (및 en/ja/zh) Navigation 섹션에 번역 키 추가:
```json
"master_orders": "마스터 포장"
```

---

## 관련 파일

| 파일 | 설명 |

| `src/components/layout/NaviSidebar.tsx` | NAV_ITEMS 수정 대상 |
