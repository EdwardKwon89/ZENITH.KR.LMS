## 현상

`/master/geo` 페이지가 구현되어 있으나 NaviSidebar에 등록되지 않아 URL 직접 입력 없이 접근 불가.

## 기능 설명

- 국가(`zen_nations`) 및 항구(`zen_ports`) 등록·수정·삭제 CRUD
- ADMIN 전용 (`requireAdmin()`)
- 경로 최적화(`getRouteOptions`)와 스케줄 자동 매핑(`selectRoute`)의 기반 데이터

## 추가 이슈: 중복 라우트

`/admin/codes/page.tsx`와 `/master/codes/page.tsx`가 **동일한 파일** (diff 결과 identical).
`/master/geo`는 별도 기능이나, `/master/*` 경로가 사이드바의 `/admin/codes` parent href와 불일치.
두 경로 체계(`/admin/*` vs `/master/*`) 정리 필요.

## 권장 조치

`NaviSidebar.tsx` "기본 정보" 그룹 하위에 추가:

```ts
{ title: t("geo_master"), href: "/master/geo" },
{ title: t("common_codes"), href: "/admin/codes" },
```

## 관련 파일

| 파일 | 설명 |
|:---|:---|
| `src/app/[locale]/(dashboard)/master/geo/page.tsx` | 지리정보 관리 페이지 |
| `src/app/[locale]/(dashboard)/master/codes/page.tsx` | 공통코드 관리 (admin/codes와 중복) |
| `src/components/layout/NaviSidebar.tsx` | NAV_ITEMS 수정 대상 |
