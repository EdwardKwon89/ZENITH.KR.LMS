## 배경

`createOrder()` 트랜잭션(IMP-019)은 완료되었으나, 그 외 주요 쓰기 작업이 여전히 트랜잭션 보호 없음.

- `updateOrderStatus()`: 상태변경·히스토리·인벤토리·정산·알림·트래킹 8회 순차 쿼리 — 중간 실패 시 데이터 불일치 발생 가능
- `payInvoiceFromWallet()`: 지갑 잔액 차감 → 인보이스 업데이트 순차 실행 — 인보이스 실패 시 잔액은 이미 차감됨

참조: `scratch/post_launch_improvements.md §IMP-047` · `src/app/actions/orders.ts` · `src/app/actions/wallet.ts`
