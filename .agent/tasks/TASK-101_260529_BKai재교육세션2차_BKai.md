### §1 — DoD 허위 체크 금지

DoD item을 `[x]`로 체크하기 전 반드시 실물 확인:
- "상태 🔔 변경" → task file line 12 실제 🔔인지 Read로 확인 후 체크
- "커밋 해시 포함" → git log로 해시 확인 후 체크
- 기대: DoD 체크는 완료의 증거이며, 허위 체크는 무결성 위반

### §2 — 재제출 시 [작업 결과] 전체 커밋 기재

원본 작업 커밋 + 재제출 커밋 전체를 [작업 결과]에 기재:
- `332a036` (원본) → `4833de1` (수정) → `9040bcc` (재제출) → `dd3644c` (해시기재) — **전량 기재**
- Aiden 반려 이후 추가된 커밋은 모두 [작업 결과]에 반영

### §3 — 완료 보고 절차 순서 엄수 (R-17 v1.5 §5)

1. 코드 커밋
2. task file [작업 결과] 기재 (모든 커밋 해시 포함) + 상태 🔔 변경
