## 구현 범위

### §A — UAT-17-01 DIRECT 배송 검증 (`tests/e2e/uat-17-01-direct-order.spec.ts`)

**시나리오**:
1. `agency_shipper@zenith.kr` 로그인
2. `/ko/orders/new` 진입 → EXP(특송) 운송 선택
3. DIRECT 기본 선택 → 픽업 입력 폼 숨김 확인 (UI 스크린샷)
4. 오더 DB insert (`delivery_method='DIRECT'`) → `pickup_location` NULL 검증
5. 테스트 데이터 정리

**검증 포인트**:
- UI: DIRECT 선택 시 pickup fields hidden
- DB: `delivery_method='DIRECT'`, `pickup_location IS NULL`

### §B — UAT-17-02 PICKUP 배송 검증 (`tests/e2e/uat-17-02-pickup-order.spec.ts`)

**시나리오**:
1. `agency_shipper@zenith.kr` 로그인
2. `/ko/orders/new` 진입 → EXP(특송) 운송 선택
3. PICKUP(픽업 수령) 선택 → 픽업 입력 폼 활성화 확인 (UI 스크린샷)
4. Step 1 필수 입력 → Step 2 서비스 조합 선택 → Step 3 요율 선택
5. 픽업 필드 비우고 submit 시도 → Zod 검증 차단 확인
6. 오더 DB insert (`delivery_method='PICKUP'` + pickup data) → 정상 적재 확인
7. 테스트 데이터 정리

**검증 포인트**:
- UI: PICKUP 선택 시 pickup fields visible
- Zod: 빈 pickup 필드로 submit 시 검증 차단
- DB: `delivery_method='PICKUP'`, `pickup_location`·`pickup_contact_name`·`pickup_contact_tel` 정상 적재
