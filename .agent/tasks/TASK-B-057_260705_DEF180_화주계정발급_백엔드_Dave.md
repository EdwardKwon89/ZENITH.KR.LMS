```typescript
import crypto from 'crypto';

function generateTempPassword(): string {
  // 혼동 문자(0/O, 1/I/l) 제외 — 12자리 영문+숫자
  const chars = 'ABCDEFGHJKLMNPQRSTUVWXYZabcdefghjkmnpqrstuvwxyz23456789';
  const bytes = crypto.randomBytes(12);
  return Array.from(bytes).map(b => chars[b % chars.length]).join('');
}
```

## ❌ 반려 사유 (2026-07-07 Jaison)

로컬 E2E 검증 중 하기 3건의 버그 발견. PR#184 코드 수정 후 재제출 요청.

### BUG-1: `_createShipperProfile` — INSERT → upsert 필수

**현상**: 화주 등록 시 `"duplicate key value violates unique constraint (zen_profiles_pkey)"` 오류 발생, 등록 실패.

**원인**: `on_auth_user_created` DB 트리거(`handle_new_user`)가 `auth.users` INSERT 시 `zen_profiles`에 자동으로 row를 생성함. 이후 `_createShipperProfile`의 INSERT가 동일한 `id`로 충돌.

**수정**: `supabase.from('zen_profiles').insert(...)` → `.upsert(...)` 로 변경.

```typescript
// BEFORE (버그)
const { error } = await supabase.from('zen_profiles').insert({ id: authUid, ... });

// AFTER (수정)
const { error } = await supabase.from('zen_profiles').upsert({ id: authUid, ... });
```

---

### BUG-2: 이메일 실패 시 등록 전체 차단

**현상**: Resend API 키 미설정 또는 네트워크 오류로 이메일 전송 실패 시, 화주 계정 자체가 생성되지 않음.

**원인**: 현재 흐름에서 ④ 이메일 실패 → `auth.admin.deleteUser` 롤백 → 에러 반환.

**수정**: 이메일을 **best-effort**로 처리. 실패해도 등록은 계속 진행하고 `inviteEmailSent: false` 반환.

```typescript
// AFTER (수정)
let inviteEmailSent = false;
try {
  await sendShipperWelcomeEmail({ ... });
  inviteEmailSent = true;
} catch (emailErr) {
  logger.warn(`[createAgencyShipper] Welcome email failed (non-fatal): ${emailErr}`);
}
// 이메일 실패 시 롤백 없이 ⑤로 진행
```

---

### BUG-3: 롤백 순서 불안전 — zen_profiles 고아 레코드 발생

**현상**: ⑥ 또는 ⑦ 단계 실패 시 롤백 후에도 `zen_profiles` row가 잔류. 재시도 시 `zen_profiles_email_key` unique constraint 위반.

**원인**: BUG-1과 동일 — `on_auth_user_created` 트리거가 `zen_profiles`를 자동 생성하지만, 현재 롤백 코드는 `auth.admin.deleteUser`만 호출하고 `zen_profiles`는 삭제하지 않음. `auth.users` 삭제 시 FK cascade도 없음.

**수정**: `_rollbackShipper` 헬퍼 추가 — **삭제 순서 반드시 zen_profiles → auth user → org** 보장.

```typescript
async function _rollbackShipper(
  supabase: SupabaseClient,
  authUid: string,
  orgId?: string,
): Promise<void> {
  const { error: profileErr } = await supabase.from('zen_profiles').delete().eq('id', authUid);
  if (profileErr) logger.warn(`[rollback] zen_profiles delete failed for ${authUid}: ${profileErr.message}`);

  const { error: authErr } = await supabase.auth.admin.deleteUser(authUid);
  if (authErr) logger.warn(`[rollback] auth.deleteUser failed for ${authUid}: ${authErr.message}`);

  if (orgId) {
    const { error: orgErr } = await supabase.from('zen_organizations').delete().eq('id', orgId);
    if (orgErr) logger.warn(`[rollback] zen_organizations delete failed for ${orgId}: ${orgErr.message}`);
  }
}
```

기존 인라인 롤백 코드(`auth.admin.deleteUser` 직접 호출) 전량을 `_rollbackShipper` 호출로 교체.
