## 구현 범위

### §1 — 임시 PW 생성 유틸 (`src/app/actions/agency/shippers.ts` 내부)

```typescript
import crypto from 'crypto';

function generateTempPassword(): string {
  // 혼동 문자(0/O, 1/I/l) 제외 — 12자리 영문+숫자
  const chars = 'ABCDEFGHJKLMNPQRSTUVWXYZabcdefghjkmnpqrstuvwxyz23456789';
  const bytes = crypto.randomBytes(12);
  return Array.from(bytes).map(b => chars[b % chars.length]).join('');
}
```

### §2 — `_createShipperProfile` 신규 내부 함수

```typescript
async function _createShipperProfile(
  supabase: SupabaseClient,
  authUid: string,
  email: string,
  fullName: string,
  orgId: string,
  phone?: string | null,
): Promise<void> {
  const { error } = await supabase.from('zen_profiles').insert({
    id: authUid,
    email,
    full_name: fullName,
    org_id: orgId,
    role: 'AGENCY_SHIPPER',
    status: 'ACTIVE',
    is_active: true,
    grade_code: 'BRONZE',
    phone_number: phone ?? null,
  });
  if (error) throw new Error(`Failed to create shipper profile: ${error.message}`);
}
```

### §4 — `createAgencyShipper` 전면 재구현

처리 흐름:
```
① 입력 유효성 검사 (login_email 필수 포함)
② generateTempPassword()
③ supabase.auth.admin.createUser({ email: login_email, password: tempPassword, email_confirm: true })
   └→ 실패 → fieldErrors 반환 (이메일 중복 등)
④ sendShipperWelcomeEmail(login_email, name, tempPassword, contact_email) via Resend
   └→ 실패 → auth.admin.deleteUser(authUid) 후 에러 반환
⑤ _createShipperOrg(supabase, ..., 주소 6개 필드)
   └→ 실패 → auth.admin.deleteUser(authUid) 후 에러 반환
⑥ _createShipperProfile(supabase, authUid, login_email, name, orgId, contact_phone)
   └→ 실패 → auth.admin.deleteUser(authUid) + org DELETE 후 에러 반환
⑦ _linkShipperToAgency(supabase, agencyOrgId, orgId, { ...data, grade: data.grade ?? 'BRONZE' })
   └→ 실패 → 위 3건 롤백
⑧ return { success: true, shipperId: linkId, inviteEmailSent: true }
```

반환 타입 변경:
```typescript
export type CreateAgencyShipperResult =
  | { success: true; shipperId: string; inviteEmailSent: boolean }
  | { success: false; fieldErrors: Record<string, string> };
```
