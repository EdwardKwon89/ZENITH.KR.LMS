## 목표

TASK-100(DEF-030 경로 최적화 전면 수정)으로 변경된 UI·업무 흐름을 UAT 절차서에 반영한다.

- **UAT-02-04**: SCHEDULED 가드 반영 — 사전 조건에 경로 선택 완료 조건 추가
- **UAT-02-09**: 3종 카드 UI → 전체 후보 비교 테이블로 절차·합격 기준 전면 수정 + PENDING 상태 오기 수정
- **UAT-10-01**: 3종 카드 → 전체 후보 비교 테이블로 절차·합격 기준 수정

---

## 배경

TASK-100에서 다음 두 가지 구조적 변경이 발생했다.

**① REGISTERED→SCHEDULED 가드 추가** (`orders.ts updateOrderStatus`):
- `route_option_id`가 NULL이면 SCHEDULED 전이 거부 + 에러 메시지 반환
- UAT-02-04 step 3(`REGISTERED→SCHEDULED`)은 사전에 경로 선택이 없으면 실패

**② RouteOptimizationSection.tsx UI 전면 교체**:
- 기존: 3종 카드(최저비용·최단시간·최적균형) grid 1×3
- 변경: 전체 후보 비교 테이블 (직항/경유 그룹, 추천 배지 — `최저비용 추천`/`최단시간 추천`/`균형 추천`)
- UAT-02-09·UAT-10-01의 절차·기대결과·합격 기준이 구 UI 기준으로 작성됨

## 작업 범위

### 수정 파일 1: `docs/91_FinalTest/UAT/UAT_02_오더관리.md`

#### [UAT-02-04] 오더 상태 전이 전체 플로우

**사전 조건 수정** — 경로 선택 완료 조건 추가:

| 현재 | 변경 후 |
|:----|:-------|
| ADMIN 계정 로그인 상태, **REGISTERED** 상태 오더 1건 존재 | ADMIN 계정 로그인 상태, **REGISTERED** 상태 오더 1건 존재, **해당 오더의 경로 선택 완료** (`route_option_id` 존재 — UAT-02-09 선행 수행 또는 오더 상세에서 사전 경로 선택) |

> 이유: TASK-100에서 REGISTERED→SCHEDULED 전이 시 `route_option_id` 필수 가드 추가. 미선택 시 "경로를 먼저 선택해야 일정 확정(SCHEDULED)이 가능합니다." 오류 반환.

#### [UAT-02-09] 경로 옵션 조회 및 선택

**사전 조건 수정**:
- `PENDING 또는 WAREHOUSED` → `REGISTERED` (PENDING은 시스템에 존재하지 않는 상태)

**테스트 절차 전면 수정** (step 3~8 재작성):

| 순서 | 현재 기대 결과 | 변경 후 기대 결과 |
|:---:|:-------------|:----------------|
| 3 | 3종 카드(최저비용·최단시간·최적균형) grid 1×3 표시 | 전체 운송사 후보 **비교 테이블** 표시 — 직항/경유 그룹 분리, 각 행에 운송사·경로(세그먼트)·운송 방식·비용·소요일·추천 배지 표시 |
| 4 | 각 카드에 Total Cost·Transit Time·구간정보·Score 표시 | 추천 배지 확인: 최저비용 행 → `최저비용 추천` 배지, 최단시간 행 → `최단시간 추천` 배지, 균형 행 → `균형 추천` 배지 |
| 5 | '최저비용' 카드 클릭 → '이 경로 선택' 버튼 | 비교 테이블 행의 **'선택'** 버튼 클릭 → "경로가 확정되었습니다" 토스트 표시, 해당 행 하이라이트 |
| 6 | 마일스톤 타임라인 표시 | (동일 유지) |
| 7 | 새로고침 후 선택 유지 | (동일 유지) |
| 8 | '경로 재계산' 버튼 → 3종 카드 다시 표시 | '경로 재계산' 버튼 → **비교 테이블** 재표시 |

**합격 기준 전면 수정**:
- `3종 경로 카드(COST·TIME·BALANCED) 정상 표시` → `전체 운송사 후보 비교 테이블 표시 (직항/경유 그룹)`
- `카드에 총비용·총소요일·구간정보·스코어 모두 표시` → `각 행에 운송사·경로·비용·소요일 표시, 비용=0인 경우 "요율 미등록" 표시`
- `경로 선택 후 토스트 및 마일스톤 타임라인 표시` → (동일 유지)
- `SHIPPER 역할에서도 경로 조회·선택 가능` → (동일 유지)
- 추가: `추천 배지(최저비용·최단시간·균형) 정상 표시`

**테스트 절차 수정** (step 3~7 재작성):

| 순서 | 현재 기대 결과 | 변경 후 기대 결과 |
|:---:|:-------------|:----------------|
| 3 | COST·TIME·BALANCED 카드 3종 표시 | 전체 운송사 후보 **비교 테이블** 표시 — 직항/경유 그룹 분리 |
| 4 | COST 카드 내용 확인 | 각 행: 운송사명·경로 세그먼트·운송 방식·비용($)·소요일·추천 배지 |
| 5 | TIME 카드 내용 확인 | 추천 배지 확인: 최저비용 행 `최저비용 추천` · 최단시간 행 `최단시간 추천` · 균형 행 `균형 추천` |

**합격 기준 전면 수정**:
- `경로 옵션 3종 (COST·TIME·BALANCED) 카드 정상 표시` → `전체 운송사 후보 비교 테이블 표시 (직항/경유 그룹 분리)`
- `BALANCED 카드가 기본 추천으로 강조 표시` → `추천 배지 3종(최저비용·최단시간·균형) 해당 행에 정상 표시`
- 추가: `zen_route_options.recommended_for 컬럼 값 정상 저장 확인`
