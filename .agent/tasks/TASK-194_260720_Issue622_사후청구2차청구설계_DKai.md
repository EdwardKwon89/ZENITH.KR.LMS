## [설계 옵션] (Aiden 정리, Edward 확인 대기)

1. **별도 2차 인보이스 신규 발행**: `zen_invoices`에 원 오더를 참조하는 별도 인보이스 행 생성(`metadata.source_order_id` + `adjustment_of: <원 invoice_id>` 등). 이미 발행된 인보이스는 불변 유지 — 회계 관행상 가장 안전.
2. **크레딧/데빗 노트**: 차액이 양수(추가청구)면 데빗노트, 음수(환급)면 크레딧노트로 별도 문서 발행. UI 문구 중 "화주 정산 차감/크레딧 발행" 표현과 일치하는 개념이나 신규 테이블/문서 타입 설계 필요.
3. **기존 인보이스 소급 수정**: 기존 `zen_invoices.total_amount`에 조정액을 반영. 구현은 가장 간단하나 이미 화주에게 전달된 인보이스 금액이 사후에 바뀌는 것이라 회계 관행상 지양 권장.

**Aiden 1차 권고**: 옵션 1(별도 2차 인보이스) — 기존 인보이스 불변 원칙 유지, `zen_order_costs.invoice_id`를 이 신규 인보이스로 연결하면 기존 스키마(정산 조회 로직)를 그대로 재사용 가능. 단, 최종 결정은 Edward 확인 후 확정.
