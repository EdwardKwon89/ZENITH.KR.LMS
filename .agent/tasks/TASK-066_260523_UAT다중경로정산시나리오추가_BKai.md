2. **UAT_02_오더관리.md 보완**: `docs/91_FinalTest/UAT/UAT_02_오더관리.md`
   - **UAT-02-09 신규 추가**: 경로 옵션 조회 및 선택
     - 역할: SHIPPER / ADMIN
     - 화면: `/ko/orders/{order_id}` 오더 상세 > 경로 옵션 섹션
     - 시나리오: 오더 생성 완료 → 경로 옵션 탭/섹션 진입 → COST·TIME·BALANCED 3종 카드 표시 확인 → 구간(segment) 정보(출발·도착·운송사·운송수단·예상일·비용) 표시 확인 → 경로 선택 버튼 클릭 → 선택 완료 토스트
     - 합격 기준: 3종 카드 정상 표시, 경로 선택 반영, 선택 후 정산 기준 경로로 확정

3. **UAT_05_정산_인보이스.md 보완**: `docs/91_FinalTest/UAT/UAT_05_정산_인보이스.md`
   - **UAT-05-08 신규 추가**: 단일 carrier 다중 구간 정산 검증
     - 역할: ADMIN
     - 전제 조건: 동일 carrier의 2-leg 경로(ICN→NRT→LAX)가 선택된 오더 1건 존재
     - 시나리오: 인보이스 생성 → 비용 상세 확인 → 전체 운임이 구간 합산으로 표시됨 (구간: ICN→NRT + NRT→LAX, carrier 동일)
     - 합격 기준: 합산 1건으로 정산, route_option 정보 인보이스에 반영
   - **UAT-05-09 신규 추가**: 다중 carrier 구간별 정산 분리 검증
     - 역할: ADMIN
     - 전제 조건: 구간별 다른 carrier (ICN→NRT: Carrier A, NRT→LAX: Carrier B)가 선택된 오더 1건 존재
     - 시나리오: 인보이스 생성 → 비용 상세 확인 → 구간별 carrier 분리 비용 표시 (Carrier A 구간비 + Carrier B 구간비)
     - 합격 기준: 구간별 carrier 분리 정산, 총액 정확

4. **UAT_MASTER.md 갱신**: `docs/91_FinalTest/UAT/UAT_MASTER.md`
   - 인덱스에 3개 행 추가 (상태 ✅, 담당 B_Kai):

   - 총 시나리오 수: 56 → 59
