## 구현 명세

### 1. `selectRoute()` 내부 보강

**파일**: TASK-100 (`5b63421`)에서 수정한 selectRoute 위치 확인 필요 (아마 `src/app/actions/orders/routing.ts` 또는 `src/app/actions/orders/route-select.ts`)

**처리 흐름**:
```
selectRoute(orderId, optionId)
  ↓ 기존 로직 — route option 확정, zen_order_routes UPSERT, carrier_id 동기화
  ↓ [신규] segments 배열에서 첫 번째 segment 추출
  ↓ [신규] zen_vessel_schedules 조회
      WHERE carrier_id = segment.carrier_id
        AND origin_port_id = segment.from_port_id (포트 코드 → UUID 역매핑 필요 시)
        AND destination_port_id = segment.to_port_id
        AND service_type = segment.transport_mode
        AND etd > NOW()
      ORDER BY etd ASC
      LIMIT 1
  ↓ [신규] 매칭 성공 → segments JSONB 업데이트
  ↓ [신규] 매칭 실패 → null 허용, 경고 로그만 (non-fatal)
```

**⚠️ 포트 ID 주의**:
- `zen_route_network.from_port_id` / `to_port_id`는 PORT **CODE** (예: "ICN") 저장
- `zen_vessel_schedules.origin_port_id` / `destination_port_id`는 PORT **UUID** 저장
- 매칭 시 `zen_ports` 테이블을 통해 CODE → UUID 변환 필요

### 2. segments JSONB 필드 추가

```typescript
// 기존 segments 구조
{
  carrier_id, transport_mode, from_port_id, to_port_id,
  transit_days, cost, currency
}

// 보강 후
{
  carrier_id, transport_mode, from_port_id, to_port_id,
  transit_days, cost, currency,
  schedule_id?: string,   // zen_vessel_schedules.id
  flight_no?: string,     // vessel_name + (voyage_no 있으면 " / " + voyage_no)
  etd?: string,           // ISO datetime
  eta?: string            // ISO datetime
}
```

### 3. UI 표시

오더 상세 페이지에서 segments에 flight_no / etd가 있으면 표시, 없으면 미표시 (기존 UI 유지).

표시 위치:
- **RouteOptionCard** (경로 선택 카드): 각 segment에 편명/ETD 표시
- **오더 상세 Tracking 패널**: SCHEDULED 상태일 때 "편명: KE081 | ETD: 2026-06-10" 형식

### 4. UAT 시드 스케줄 등록

ADMIN이 UAT 실행 전 `/admin/schedules`에서 아래 조합 스케줄 등록 필요. D_Kai는 `seed-local.ts`에 `seedVesselSchedules()` 추가하여 자동 등록:

| Carrier | Service | Origin | Dest | ETD (예시) |
|:-------:|:-------:|:------:|:----:|:----------:|
| KAL (또는 시드에 있는 AIR carrier) | AIR | ICN | JFK | 2026-07-01 |
| HMM (또는 시드에 있는 SEA carrier) | SEA | ICN | JFK | 2026-07-05 |
| 기존 시드 carrier | AIR | ICN | LAX | 2026-07-02 |

실제 carrier UUID는 `zen_carriers` 시드 데이터 기준으로 조회하여 사용.
