## 배경

리스트 조회 엔드포인트 18곳에서 페이지네이션 없이 전체 데이터를 반환합니다.
대량 데이터 환경에서 응답 지연, 메모리 초과, 클라이언트 렌더링 블로킹이 발생합니다.
`limit` / `offset` 또는 커서 기반 페이지네이션 적용이 필요합니다.

---

## 수정 대상 (18곳)

`gitnexus_query({query: "list query without pagination limit"})` 결과로 전수 파악

예상 패턴:
```typescript
// 현재 (위험)
const { data } = await supabase.from('zen_orders').select('*')

// 목표
const { data } = await supabase
  .from('zen_orders')
  .select('*')
  .range(offset, offset + limit - 1)  // 또는 .limit(pageSize)
```
