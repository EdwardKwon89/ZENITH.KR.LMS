## 설계 의견 (Agent 작성)

### 방안 A (제안) — 기존 패턴 재사용

**1. PDF 라이브러리**: `@react-pdf/renderer` (기존 사용 중, v4.5.1)
- 근거: PackingListPDF.tsx·TaxInvoiceTemplate.tsx에서 동일 라이브러리 사용 중
- CJK 폰트(Noto Sans KR) 이미 등록 완료, A4 레이아웃 패턴 확립
- ShippingLabelPDF.tsx 신규 생성 → PDFDownloadLink로 다운로드

**2. 출고 처리 상태 관리**
- `zen_inventory` 모델은 quantity 기반(on_hand_qty·reserved_qty·available_qty) — 별도 STORED/SHIPPED enum 불필요
- 출고 확정 시: `available_qty` 차감 + `InventoryHistory`(type='OUTBOUND') 기록
- `confirmOutbound(inventoryIds[], quantities[])` Server Action 신규 구현

**3. 신규 라우트**: `src/app/[locale]/(dashboard)/warehouse/outbound/page.tsx`
- 화면 구성: 출고 대상 재고 목록 (available_qty > 0) → 개별 선택/수량 입력 → 운송장 PDF 미리보기 → 출고 확정

**4. 운송장 PDF 레이아웃**: A4 1페이지
- 헤더: ZENITH LMS 로고 + "운송장 (SHIPPING LABEL)" 타이틀 + 바코드
- 섹션: 송하인 정보 → 수하인 정보 → 오더 참조 → 품목 리스트(description·qty·weight) → 총계
- 하단: 생성 일시 + 시스템 풋터

**5. NaviSidebar**: `warehouse` 그룹에 "출고 처리" 메뉴 추가 (기존 재고 관리 아래 위치)

## 설계 확정 (Aiden 작성)

**확정 (2026-05-23, Aiden)** — B_Kai 방안 A 부분 채택, 출고 처리 방식 수정.

**채택 사항:**
- **PDF 라이브러리**: ✅ `@react-pdf/renderer` 채택 — ShippingLabelPDF.tsx 신규 생성, PDFDownloadLink 패턴 유지
- **NaviSidebar**: ✅ `warehouse` 그룹에 "출고 처리" 메뉴 추가

**수정 지시 — 출고 처리 방식:**

> ⚠️ B_Kai 제안의 `available_qty 차감 + InventoryHistory` 방식은 부적합. `zen_inventory`에는 STORED/SHIPPED status 컬럼 없음. 아래로 변경 필수.

**확정 방식: WAREHOUSED 오더 기반**
- **출고 대상 목록**: `zen_orders WHERE status = 'WAREHOUSED'` 조회 (inventory qty 기반 아님)
- **운송장 PDF**: 오더 정보(수하인·품목·바코드·오더번호) 기반 A4 레이아웃 — ShippingLabelPDF.tsx
- **출고 확정 Server Action**: `confirmOutbound(orderId: string)`
  1. `updateOrderStatus(orderId, OrderStatus.RELEASED, '[출고확정]')` — WAREHOUSED→RELEASED
  2. `zen_inventory_history` INSERT (transaction_type='OUTBOUND', reference_id=orderId, change_qty=-qty)
- **오늘의 출고 이력**: `order_status_history WHERE next_status='RELEASED'` AND 오늘 날짜

**착수 승인**: 🔍→🔄 착수 가능. 수정 사항 반영 후 구현.
