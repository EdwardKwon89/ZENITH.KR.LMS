**수정 지시 — 출고 처리 방식:**

> ⚠️ B_Kai 제안의 `available_qty 차감 + InventoryHistory` 방식은 부적합. `zen_inventory`에는 STORED/SHIPPED status 컬럼 없음. 아래로 변경 필수.

**확정 방식: WAREHOUSED 오더 기반**
- **출고 대상 목록**: `zen_orders WHERE status = 'WAREHOUSED'` 조회 (inventory qty 기반 아님)
- **운송장 PDF**: 오더 정보(수하인·품목·바코드·오더번호) 기반 A4 레이아웃 — ShippingLabelPDF.tsx
- **출고 확정 Server Action**: `confirmOutbound(orderId: string)`
  1. `updateOrderStatus(orderId, OrderStatus.RELEASED, '[출고확정]')` — WAREHOUSED→RELEASED
  2. `zen_inventory_history` INSERT (transaction_type='OUTBOUND', reference_id=orderId, change_qty=-qty)
- **오늘의 출고 이력**: `order_status_history WHERE next_status='RELEASED'` AND 오늘 날짜

**착수 승인**: 🔍→🔄 착수 가능. 수정 사항 반영 후 구현.
