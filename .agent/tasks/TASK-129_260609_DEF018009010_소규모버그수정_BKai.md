## 작업 범위

### §1 — DEF-018: CARRIER 등록 시 role=ADMIN 코드 수정

**문제**: `login/actions.ts` 신규 조직 생성자 role이 `org_type` 무관하게 항상 `ADMIN`으로 고정됨.

**현재 코드** (`login/actions.ts:146` 부근):
```typescript
role: isNewOrg ? USER_ROLES.ADMIN : ...
```

**수정 방향**:
- 신규 조직 생성 시 `org_type`에 따라 role 결정:
  - `PLATFORM` 조직 → `ADMIN`
  - `CARRIER` 조직 → `CARRIER`
  - `SHIPPER` / 기타 → 기존 로직 유지
- `app_metadata` 동기화도 함께 수정 (`supabase.auth.admin.updateUserById`)

**⚠️ 주의**: 기존 `test_uat01@zenith.kr`은 이미 데이터 조치 완료 (role CARRIER로 변경). 이번 작업은 코드 수정만.

### §2 — DEF-009: 증빙서류 오딧 버튼 오류

**문제**: 법인 심사 화면에서 "증빙서류 오딧" 버튼 클릭 시 "파일을 불러올 수 없습니다." 메시지 표출.

**확인 항목**:
- 파일 조회 server action 또는 API 경로 확인
- RLS 정책 또는 Storage bucket 접근 권한 확인
- 수정 후 동작 확인

### §3 — DEF-010: 법인 최종 승인 버튼 글자 안 보임

**문제**: "법인 최종 승인" 버튼의 글자(텍스트)가 보이지 않음.

**수정 방향**: CSS variant 또는 텍스트 색상 문제. 버튼 스타일 확인 후 수정.
