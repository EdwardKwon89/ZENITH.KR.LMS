## 목표

`fn_get_best_matching_rate` DB 함수가 `p_origin_port`·`p_dest_port` 파라미터를 받지만 WHERE 절에서 무시하고 있어, 동일 운송사+모드인 요율 카드가 항로 무관하게 동일하게 반환되는 문제를 해결한다.

- `zen_rate_cards`에 `origin_port_id` / `dest_port_id` 컬럼 추가
- `fn_get_best_matching_rate` WHERE 절에 포트 조건 추가 (정밀 우선 · NULL fallback)
- `TISARateMatcher.matchRateCard()` 포트 파라미터 지원
- admin/rates UI에 origin/dest port 드롭다운 추가

---

## 배경

### 문제

`fn_get_best_matching_rate`(`carrier_id`, `transport_mode`, `p_origin_port`, `p_dest_port`, ...) 시그니처가 존재하나 WHERE 절은 다음만 사용:

```sql
WHERE rc.carrier_id = p_carrier_id
  AND rc.transport_mode = p_transport_mode
  AND rc.is_active = TRUE
  AND rc.valid_from <= NOW()
  AND (rc.valid_until IS NULL OR rc.valid_until >= NOW())
```

`p_origin_port`·`p_dest_port`가 완전히 무시됨 → 같은 carrier+mode면 항로 불문 동일 요율 카드 반환.

`zen_rate_cards` 테이블에도 `origin_port_id`·`dest_port_id` 컬럼 없음 → 포트 지정 자체가 불가.

### 영향

- TISA 요율 산정 시 ICN→SIN과 ICN→LAX가 동일 요율 적용
- UAT-11-01/02 FAIL의 간접 원인 (정확한 항로별 요율이 반환되지 않음)

## 설계 확정 (Aiden)

다음 설계를 확정한다. D_Kai는 아래 기준을 준수하여 구현한다.

### 1. 포트 컬럼 (zen_rate_cards)

| 컬럼 | 타입 | 제약 | 의미 |
|:----|:----|:----|:----|
| `origin_port_id` | `UUID` | `NULL`, FK `zen_ports(id)` | NULL = 출발항 무관 |
| `dest_port_id` | `UUID` | `NULL`, FK `zen_ports(id)` | NULL = 도착항 무관 |

**NULL 의미**: 해당 포트를 지정하지 않은 요율 카드는 **모든 항로에 적용 가능한 fallback 카드**.

### 2. 매칭 우선순위 (정밀 우선 · NULL fallback)

```sql
ORDER BY
  (CASE WHEN rc.origin_port_id = p_origin_port THEN 1 ELSE 0 END +
   CASE WHEN rc.dest_port_id = p_dest_port THEN 1 ELSE 0 END) DESC,
  rc.valid_from DESC
LIMIT 1;
```

- origin+dest 모두 일치 (score 2) > 한 쪽만 일치 (score 1) > 둘 다 NULL (score 0)
- p_origin_port / p_dest_port가 NULL이면 모든 카드 대상

### 3. WHERE 절 추가 조건

```sql
AND (rc.origin_port_id IS NULL OR rc.origin_port_id = p_origin_port)
AND (rc.dest_port_id IS NULL OR rc.dest_port_id = p_dest_port)
```

### 4. TISARateMatcher 시그니처

```typescript
matchRateCard(carrierId: string, mode: string, originPortId?: string, destPortId?: string)
```

호출 측(`tisa.ts`)에서 `zen_orders.origin_port_id` / `zen_orders.dest_port_id`를 읽어 전달.

### 5. Admin UI

- `RateCardForm`에 origin port / dest port 드롭다운 추가 (optional)
- 선택 없음 = NULL 저장 (항로 무관 fallback 카드)
- `useRates.ts`에 `originPortId` / `destPortId` 상태 추가
