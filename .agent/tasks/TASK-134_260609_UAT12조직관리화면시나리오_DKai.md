## 작업 범위

### §1 — UAT-12 절차서 신규 작성

**파일**: `docs/91_FinalTest/UAT/UAT_12_조직관리화면.md`

**작성 기준**:
- 기존 UAT 절차서(UAT_01~11) 형식 준수
- 각 시나리오: 사전조건 · 단계별 절차 · 기대 결과
- 참조 기능: `src/app/[locale]/(dashboard)/admin/organizations/manage/`

**UAT-12-01: CUSTOMS 조직 신규 등록**
```
사전조건: ADMIN 계정 로그인
STEP1: 사이드바 > 조직 관리 > [조직 관리] 메뉴 클릭
STEP2: /admin/organizations/manage 접속 확인
STEP3: [신규 등록] 버튼 클릭 → 등록 모달 표시 확인
STEP4: 조직명, 유형(CUSTOMS), 사업자번호, 대표자명 입력
STEP5: 등록 버튼 클릭 → 목록에 신규 조직 표시 확인 (ACTIVE 상태)
기대결과: zen_organizations에 CUSTOMS 타입 레코드 생성
```

**UAT-12-02: DELIVERY 조직 신규 등록**
```
사전조건: ADMIN 계정 로그인
STEP1~5: UAT-12-01과 동일하되 유형 = DELIVERY
기대결과: zen_organizations에 DELIVERY 타입 레코드 생성
```

**UAT-12-03: 조직 상태 변경 (ACTIVE → SUSPENDED)**
```
사전조건: ACTIVE 상태 조직 1건 존재
STEP1: 조직 관리 목록에서 대상 조직 확인
STEP2: 상태 토글(또는 액션 버튼) 클릭 → SUSPENDED로 변경
STEP3: 목록에서 상태 SUSPENDED 배지 확인
기대결과: zen_organizations.status = 'SUSPENDED' 반영
```

**UAT-12-04: 조직 목록 필터링**
```
사전조건: CUSTOMS·DELIVERY·CARRIER 조직이 각 1건 이상 존재
STEP1: 유형 필터 > CUSTOMS 선택 → 목록에 CUSTOMS 조직만 표시 확인
STEP2: 유형 필터 초기화 > 상태 필터 > ACTIVE 선택 → ACTIVE 조직만 표시 확인
STEP3: 검색어 입력 → 조직명 포함 항목만 표시 확인
기대결과: 필터·검색 동작 정상
```
