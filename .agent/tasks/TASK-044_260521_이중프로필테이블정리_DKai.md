## 배경

`profiles`(Supabase Auth 초기 스키마)와 `zen_profiles`(ZENITH 리팩토링) 두 테이블이 공존한다.
현재 코드는 두 테이블을 모두 업데이트하거나 병합 조회하는 이중 로직을 포함한다.

- **`updateMyProfile()`**: 두 테이블 모두 업데이트
- **`getCurrentUserAffiliation()`**: 두 테이블 조회 후 병합
- **위험**: 두 테이블 간 데이터 불일치 가능, RLS 정책 이중화

참조: `scratch/post_launch_improvements.md §IMP-049` · `src/types/supabase.ts` · `src/app/actions/member.ts` · `supabase/migrations/*.sql`
