### Rollback 계획
```sql
-- rollback: profiles 복원 (M1-M8 역순)
CREATE TABLE profiles (LIKE zen_profiles);
INSERT INTO profiles SELECT * FROM zen_profiles;
ALTER TABLE profiles ADD COLUMN grade_code TEXT;
UPDATE profiles p SET grade_code = zp.grade_code FROM zen_profiles zp WHERE p.id = zp.id;
```

| 항목 | 내용 |
|:-----|:------|
| **제안 방안** | **방안 B** — `profiles` 완전 삭제·`zen_profiles`에 `grade_code` 추가 |
| **선택 근거** | `profiles` 의존 5곳만 수정, Auth Trigger 불필요(이미 `zen_profiles` INSERT ✅), 기존 리팩토링 방향 일치 |
| **예상 리스크** | LOW — rollback 스크립트로 완전 복원 가능. 단 `DROP TABLE` 전 데이터 백업 필수 |
| **대안 방안** | • **방안 A** (VIEW): Auth trigger가 `profiles`에 INSERT 시 VIEW로는 INSERT 불가 → `INSTEAD OF` trigger 필요 (복잡도↑) <br>• **방안 C** (역방향): `zen_profiles` 고유 컬럼 다수 소실 → ❌
