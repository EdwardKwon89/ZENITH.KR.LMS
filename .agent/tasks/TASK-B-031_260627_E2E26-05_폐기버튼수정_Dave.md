## 업무 개요

E2E-26-05 (폐기 버튼 → confirm dialog → 폐기 완료) 테스트가 실패하는 원인을 수정한다.

### 현재 증상
- `page.getByRole('button', { name: /폐기/ }).first()` 가 timeout(15s) 내에 보이지 않음
- HISTORY 패널 E2E26 카드에 "UPS 발급 완료" 배지와 "폐기" 버튼이 없음 (label info null)

### 확인된 버그 (수정 필수)

**`src/app/actions/operations/ups-labels.ts` — `fetchAndSaveLabel` 함수 (line 182~188)**

`zen_ups_labels` 테이블에 존재하지 않는 `label_data` 컬럼을 UPDATE 쿼리에 포함하고 있어 UPDATE 전체가 실패함.

```typescript
// 현재 (버그):
const { error } = await supabase
  .from('zen_ups_labels')
  .update({
    label_data: labelData,   // ← 컬럼 없음! 스키마에 미정의
    label_format: labelFormat,
    storage_path: labelUrl,
  })
  .eq('package_id', packageId);

// 수정 후:
const { error } = await supabase
  .from('zen_ups_labels')
  .update({
    label_format: labelFormat,
    storage_path: labelUrl ?? '',
  })
  .eq('package_id', packageId);
```

### 스키마 확인 근거

`supabase/migrations/20260626000000_ups_008_labels_tracking_shxk_map.sql` —  
`zen_ups_labels` 컬럼: `id, order_id, package_id, reference_no, tracking_number, label_format, storage_path, file_size_bytes, generated_at, generated_by, is_voided, voided_at, created_at`  
→ `label_data` 컬럼 **없음**.
