## 목표

요율 관리 Admin 페이지 3개의 중복·단절 구조를 3단계로 정리하여 `/admin/rates`를 단일 진실 공급원으로 확립한다.

---

## 배경 및 분석

UAT 진행 중 Admin 요율 관련 페이지 3개가 혼재함을 발견.

| 페이지 | 관리 테이블 | TISA 연결 | 문제 |
|:---|:---|:---:|:---|
| `/admin/rates` | `zen_rate_cards` | ✅ | 정상 — 포트 매칭·3계층·AdminRepository |
| `/admin/rate-cards` RateCardsTab | `zen_rate_cards` | ✅ | ⚠️ **중복** — 포트 필드 없음 |
| `/admin/rate-cards` **SurchargesTab** | **`zen_surcharges`** | ✅ | **유일한 관리 UI — 제거 전 이전 필수** |
| `/admin/transport-costs` | `zen_transport_costs` | ❌ | TISA 단절 — 운임 계산 미반영 |

**핵심 제약**: `composite-pricing.ts`가 `zen_surcharges`를 직접 조회하므로, SurchargesTab 기능을 `/admin/rates`로 먼저 이전한 뒤 `/admin/rate-cards`를 제거해야 한다.

---
