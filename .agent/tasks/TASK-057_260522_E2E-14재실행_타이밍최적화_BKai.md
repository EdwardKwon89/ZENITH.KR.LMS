## 배경

TASK-053(E2E-14 spec 작성)은 ✅ PASS됐으나, 케이스 A/B는 TASK-056 시드 데이터 부재로 SKIP됐다.
TASK-056이 ✅ 완료되어 시드 데이터(E2E-SEED-004·005 WAREHOUSED)가 준비됐으므로 재실행이 가능하다.

추가로 Aiden 분석 결과, `e2e-14-returned-flow.spec.ts`에 `waitForTimeout(3000)` × 4회 (총 12,000ms)가
존재하여 불필요한 고정 대기로 테스트 실행 시간이 과도하게 길어지는 문제가 확인됐다.

## 작업 지시

1. **[필수] E2E-14 spec 타이밍 최적화** — `tests/e2e/e2e-14-returned-flow.spec.ts` 수정

   Aiden 분석 결과, `waitForTimeout(3000)` × 4회 = 12,000ms 고정 대기 확인.
   아래 지시대로 **반드시 수정 후 실행**.

   | 위치 | 현재 코드 | 수정 방향 |
   |:-----|:---------|:---------|
   | L54 | `waitForTimeout(3000)` | `await expect(page.locator('span:has-text("RETURNED")')).toBeVisible({timeout:10000})` |
   | L86 | `waitForTimeout(3000)` | `await expect(page.locator('span:has-text("WAREHOUSED")')).toBeVisible({timeout:10000})` |
   | L134 | `waitForTimeout(3000)` | `await expect(page.locator('span:has-text("RETURNED")')).toBeVisible({timeout:10000})` |
   | L154 | `waitForTimeout(3000)` | `await expect(page.locator('span:has-text("DISPOSED")')).toBeVisible({timeout:10000})` |

   > **근거**: `waitForTimeout`은 실제 렌더링 완료 여부와 무관하게 3s를 고정 대기.
   > Playwright `expect().toBeVisible(timeout)` smart polling으로 대체하면 렌더링 즉시 통과하여
   > 전체 실행 시간을 최대 12s 단축할 수 있다.
