## 배경

화주가 경로를 선택하고 오더를 생성하면, 해당 오더에 **선택된 경로의 세그먼트 정보**가 저장되어야 한다. 이 정보는 다음 단계에서 활용된다:
- TASK-092 (IMP-086): 세그먼트별 캐리어 요율 정산
- TASK-093 (IMP-087): 환적 상태 추적 (어느 레그가 현재 진행 중인지 판단)

**현재 상태**: `zen_orders` 테이블에 선택된 경로 정보를 저장하는 필드가 없음. `zen_route_options.segments[]` JSONB가 별도 존재하나 오더와 연결되지 않음.

2. **설계 의견 제출 필수** (DB 모델 결정 필요):

   검토 필요 사항:
   - **방안 A** — `zen_orders`에 `route_option_id UUID FK` 추가 → `zen_route_options` 조인
     - 장: 정규화, 단: JOIN 추가
   - **방안 B** — `zen_orders`에 `route_segments JSONB` 직접 저장
     - 장: 조인 없이 조회, 단: 비정규화
   - **방안 C** — `zen_orders`에 `via_port_ids UUID[]` 배열 저장 (경유지 포트 ID만)
     - 장: 단순, 단: 세그먼트별 carrier/비용 정보 없음

3. **설계 확정 후 구현**:

   **a. DB 마이그레이션** — 확정된 방안으로 `zen_orders` 컬럼 추가

   **b. RLS 정책 업데이트** (신규 컬럼 접근 권한)

   **c. `createOrder()` Server Action 수정**

   **d. 오더 상세 조회 업데이트**
