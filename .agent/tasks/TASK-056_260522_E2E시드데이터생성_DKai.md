## 작업 지시

1. **본 파일 상태 → 🔄, ACTIVE_TASK.md TASK-056 → 🔄 동시 반영**
2. **`scripts/seed-local.ts` 확장**: 기존 유저/조직 시드 이후에 오더 생성 추가

   **오더 스키마 참조**: `zen_orders` 테이블
   - 필수 컬럼: `shipper_id`, `consignee_name`, `consignee_phone`, `status`, `org_id`
   - `shipper_id`: `shipper@zenith.kr` 유저 ID 사용 (기존 시드에서 생성됨)
   - `org_id`: `Global Shipper Corp` 조직 ID 사용

   **생성 대상**:
   - **PENDING 오더 5건** — E2E 전 시나리오 공용
     - 수하인 정보: `Test Consignee 1~5` / 전화: `010-1234-000N`
     - 상태: `PENDING`
     - `idempotent` 처리 필수: 이미 동일 consignee_name 오더가 있으면 skip
   - 총 5건으로 E2E-13(1건), E2E-14(2건), E2E-15(2건) 커버

3. **함수 구조**: 기존 `createUser`, `getOrCreateOrg`와 같은 방식으로 `seedOrders(supabase, shipperId, orgId)` 함수 추가

   ```typescript
   // 예시 구조
   async function seedOrders(supabase: any, shipperId: string, orgId: string) {
     const testOrders = [
       { consignee_name: 'E2E-Test Consignee 1', consignee_phone: '010-1234-0001', status: 'PENDING' },
       // ... 5건
     ];
     for (const order of testOrders) {
       const { data: existing } = await supabase.from('zen_orders')
         .select('id').eq('consignee_name', order.consignee_name).maybeSingle();
       if (existing) { console.log(`  - Order exists: ${order.consignee_name}`); continue; }
       await supabase.from('zen_orders').insert({ ...order, shipper_id: shipperId, org_id: orgId });
       console.log(`  - Order created: ${order.consignee_name}`);
     }
   }
   ```

   > ⚠️ **실제 `zen_orders` 컬럼 스키마 확인 필수**: `NOT NULL` 컬럼 누락 시 INSERT 실패. `supabase db diff` 또는 마이그레이션 파일에서 스키마 확인 후 작성.

4. **시드 실행 검증**: `rtk npx tsx scripts/seed-local.ts` 후 오더 5건 생성 확인

5. **회귀 테스트**: `rtk npm run test:regression`
