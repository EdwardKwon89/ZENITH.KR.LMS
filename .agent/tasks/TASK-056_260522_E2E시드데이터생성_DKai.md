## 배경

E2E-13/14/15 신규 시나리오 실행 중, `scripts/seed-local.ts`가 유저/조직 데이터만 생성하고
**오더 데이터를 생성하지 않음**이 확인되었다. (Noah/Codex — TASK-054 실행 시 발견)

각 테스트에 필요한 데이터:

| E2E | Task | 필요 데이터 |

| E2E-13 | TASK-052 | PENDING 오더 1건 (WAREHOUSED → HELD → 복구 시나리오) |
| E2E-14 | TASK-053 | PENDING 오더 2건 이상 (RETURNED 전이 시나리오) |
| E2E-15 | TASK-054 | PENDING 오더 2건 + 마스터 오더 생성 가능 상태 |

TASK-054(E2E-15)는 **본 Task 완료 후** 🚫→⬜ 전환됨.

## 작업 지시

1. **본 파일 상태 → 🔄, ACTIVE_TASK.md TASK-056 → 🔄 동시 반영**
2. **`scripts/seed-local.ts` 확장**: 기존 유저/조직 시드 이후에 오더 생성 추가

   **실제 `zen_orders` 스키마** (마이그레이션 검증 완료):
   - NOT NULL 필수: `order_no TEXT UNIQUE`, `cargo_details JSONB`
   - `shipper_id UUID` → `zen_organizations(id)` 참조 (`Global Shipper Corp` org ID)
   - `recipient_name TEXT` / `recipient_phone TEXT` (consignee_name/phone 없음)
   - `status TEXT` DEFAULT 'REGISTERED' — **'PENDING' 상태값 없음**
   - `org_id` 컬럼 없음 (주의: zen_orders에 org_id 존재하지 않음)

   **생성 대상 (E2E 시나리오별 상태 분류)**:
   - **REGISTERED 오더 3건** — E2E-13·E2E-15용 (초기 상태 오더)
     - `order_no`: `E2E-SEED-001` ~ `E2E-SEED-003`
     - `recipient_name`: `E2E Test Recipient 1~3`
     - `recipient_phone`: `010-9999-000N`
     - `cargo_details`: `{"description": "E2E test cargo", "weight_kg": 1.0}`
   - **WAREHOUSED 오더 2건** — E2E-14용 (RETURNED 전이는 WAREHOUSED 또는 IN_TRANSIT에서만 가능)
     - `order_no`: `E2E-SEED-004` ~ `E2E-SEED-005`
     - `status`: `'WAREHOUSED'`
     - 동일 cargo_details 구조
   - `idempotent` 처리 필수: `order_no` 기준으로 이미 있으면 skip

3. **함수 구조**: `seedOrders(supabase, shipperOrgId)` 함수 추가

   ```typescript
   async function seedOrders(supabase: any, shipperOrgId: string) {
     const testOrders = [
       { order_no: 'E2E-SEED-001', status: 'REGISTERED', recipient_name: 'E2E Test Recipient 1', recipient_phone: '010-9999-0001' },
       { order_no: 'E2E-SEED-002', status: 'REGISTERED', recipient_name: 'E2E Test Recipient 2', recipient_phone: '010-9999-0002' },
       { order_no: 'E2E-SEED-003', status: 'REGISTERED', recipient_name: 'E2E Test Recipient 3', recipient_phone: '010-9999-0003' },
       { order_no: 'E2E-SEED-004', status: 'WAREHOUSED', recipient_name: 'E2E Test Recipient 4', recipient_phone: '010-9999-0004' },
       { order_no: 'E2E-SEED-005', status: 'WAREHOUSED', recipient_name: 'E2E Test Recipient 5', recipient_phone: '010-9999-0005' },
     ];
     const cargoDetails = { description: 'E2E test cargo', weight_kg: 1.0 };
     for (const order of testOrders) {
       const { data: existing } = await supabase.from('zen_orders')
         .select('id').eq('order_no', order.order_no).maybeSingle();
       if (existing) { console.log(`  - Order exists: ${order.order_no}`); continue; }
       const { error } = await supabase.from('zen_orders').insert({
         ...order, shipper_id: shipperOrgId, cargo_details: cargoDetails
       });
       if (error) console.error(`  - Failed: ${order.order_no}`, error.message);
       else console.log(`  - Created: ${order.order_no} [${order.status}]`);
     }
   }
   ```

   > ⚠️ **E2E-14 spec 설계 주의**: `tests/e2e/e2e-14-returned-flow.spec.ts`는 목록 첫 번째 오더에서 직접 RETURNED 선택을 시도. WAREHOUSED 오더(E2E-SEED-004/005)가 목록 상단에 위치해야 시나리오가 동작함. 시드 후 해당 오더 정렬 확인 필요.

4. **시드 실행 검증**: `rtk npx tsx scripts/seed-local.ts` 후 오더 5건 생성 확인
