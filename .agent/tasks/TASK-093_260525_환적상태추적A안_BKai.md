## 배경

**A안 확정 (Edward, 2026-05-25)**: 경유지는 단순 환적(Transit) — 인천 같은 경유지에서 창고 입고/출고 오퍼레이션 없음. SCR-040/041 미적용. 상태 추적(Tracking Events)만으로 처리.

**목표**: 화주와 운영자가 오더의 현재 레그(어느 구간을 이동 중인지)를 실시간으로 파악할 수 있어야 한다.

**현재 상태**:
- `getTrackingEvents()` 함수 존재
- 레그별 컨텍스트(segment_index, hub_port) 없음
- Hub 경유 상태를 표현하는 상태값/이벤트 없음

2. **현행 파악**:
   - Tracking Event 테이블/타입 확인 (`zen_tracking_events` 또는 유사 구조)
   - 오더 상태 기계 현재 상태값 목록 확인 (`status_machine.ts` 또는 유사)

3. **Tracking Event 확장**:
   ```typescript
   // 새 이벤트 타입 추가
   'TRANSIT_DEPARTED'   // 경유지 출발 (origin 또는 hub 출발)
   'TRANSIT_ARRIVED_HUB' // 허브 환적 지점 도착
   'TRANSIT_DEPARTED_HUB' // 허브 환적 완료 — 다음 레그 출발
   'TRANSIT_ARRIVED_DEST' // 최종 목적지 도착
   ```

   이벤트에 `segment_index` (레그 번호) 및 `hub_port_code` 메타데이터 추가.

4. **오더 상태 추적 UI 업데이트**:
   - 오더 상세 페이지에서 레그별 진행 상태 표시
