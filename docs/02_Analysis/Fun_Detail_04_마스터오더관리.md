# 4. 마스터오더관리
**설명**: 여러 오더를 하나의 마스터오더로 묶어 일괄 운송 처리. 마스터오더 번호는 시스템 자동 생성(예: MO-20260415-00001).
**Phase**: Phase 2

## 4.1 마스터오더 등록
**상세**: 마스터오더를 신규 생성. 초기 상태는 CREATED. 생성 시 마스터오더번호가 자동 부여되며 이후 오더를 Packing하여 추가.
**권한**: 운영자/관리자
**Phase**: Phase 2

## 4.2 오더 Packing
**상세**: 등록된 오더를 마스터오더에 포함. Packing된 오더는 단독 수정·삭제 불가. 마스터오더 단위로 창고 입고·출고·Tracking 처리.
**권한**: 운영자/관리자
**Phase**: Phase 2

## 4.3 마스터오더 수정
**상세**: 마스터오더 상태 및 포함된 오더 구성 수정. (비고: 입고 이후 처리 단계에서는 수정 불가)
**권한**: 운영자/관리자
**Phase**: Phase 2

## 4.4 마스터오더 삭제
**상세**: 마스터오더 삭제. 입고 전 단계에서만 삭제 가능. 삭제 시 포함된 오더의 master_order_id를 Null 처리하여 오더 자체는 유지.
**권한**: 운영자/관리자
**Phase**: Phase 2

## 4.5 마스터오더 조회
**권한**: 회원/운영자/관리자
**Phase**: Phase 2
