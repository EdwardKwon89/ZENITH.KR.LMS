# BF-02 ZENITH_LMS 전체 업무 흐름 · Action 주체 · 구현 기능 매핑

> **문서번호:** BF-02
> **작성자:** Aiden (Claude, ZEN_CEO)
> **작성일:** 2026-06-10
> **버전:** v1.0
> **대상:** Edward Kwon (ZEN_CEO) 검토용

---

## 1. 개요

본 문서는 ZENITH_LMS 물류 플랫폼의 **전체 업무 흐름(End-to-End Business Flow)**을 정의하고, 각 단계별 **Action 주체**, **구현 기능 Mapping** 현황을 기록한다.

Phase 1~5 및 Phase 6 핵심 기능(다중 서비스 요율 등록) 구현 완료 후 실제 코드베이스를 기반으로 작성한 현황 기준 문서이다.

---

## 2. 전체 업무 흐름도

```
┌─────────────────────────────────────────────────────────────────────┐
│  STAGE 0. 시스템 기초정보 설정 (사전 조건)                            │
│  운임요율 / 통관요율 / 배송요율 / 스케줄 / 시스템파라미터 등록          │
└──────────────────────────────┬──────────────────────────────────────┘
                               │
┌──────────────────────────────▼──────────────────────────────────────┐
│  STAGE 1. 회원 가입 및 조직 승인                                      │
│  개인/법인 가입 신청 → 서비스 파트너 가입 → 관리자 승인 → 역할 배정     │
└──────────────────────────────┬──────────────────────────────────────┘
                               │
┌──────────────────────────────▼──────────────────────────────────────┐
│  STAGE 2. 운송 요청 등록 (화주)                                       │
│  화물정보 입력 → 서비스 선택 → 요율 확인(견적) → 제출                  │
└──────────────────────────────┬──────────────────────────────────────┘
                               │
┌──────────────────────────────▼──────────────────────────────────────┐
│  STAGE 3. 접수 확인 · 경로 배정 · 스케줄 매핑 (플랫폼 운영자)          │
│  오더 확인 → 경로 최적화(COST/TIME/BALANCED) → 선택 → 스케줄 자동매핑  │
└──────────────────────────────┬──────────────────────────────────────┘
                               │
┌──────────────────────────────▼──────────────────────────────────────┐
│  STAGE 4. 창고 입고 (창고 운영자)                                     │
│  화물 픽업·도착 → 바코드 스캔(GS1-128) → 입고 확정                    │
└──────────────────────────────┬──────────────────────────────────────┘
                               │
┌──────────────────────────────▼──────────────────────────────────────┐
│  STAGE 5. 마스터 포장 · 출고 (창고 운영자)                            │
│  하우스 오더 묶음 → 마스터오더 생성 → 출고 스캔                        │
└──────────────────────────────┬──────────────────────────────────────┘
                               │
┌──────────────────────────────▼──────────────────────────────────────┐
│  STAGE 6. 수출 통관 (통관사)                                          │
│  수출 신고 → 통관 신고번호 입력 → 반출 확인                            │
└──────────────────────────────┬──────────────────────────────────────┘
                               │
┌──────────────────────────────▼──────────────────────────────────────┐
│  STAGE 7. 운송 중 · 실시간 트래킹 (시스템 자동)                        │
│  외부 API 폴링 → 트래킹 갱신 → 화주 알림                              │
└──────────────────────────────┬──────────────────────────────────────┘
                               │
┌──────────────────────────────▼──────────────────────────────────────┐
│  STAGE 8. 수입 통관 (통관사)                                          │
│  수입 신고 → 통관 번호 확정 → 화물 반출                               │
└──────────────────────────────┬──────────────────────────────────────┘
                               │
┌──────────────────────────────▼──────────────────────────────────────┐
│  STAGE 9. 현지 배송 (배송사)                                          │
│  Last Mile 배송 접수 → 배송 진행 → 배송 완료                           │
└──────────────────────────────┬──────────────────────────────────────┘
                               │
┌──────────────────────────────▼──────────────────────────────────────┐
│  STAGE 10. 정산 · 청구 (플랫폼 운영자 / 화주)                         │
│  확정 요금 산출 → 인보이스 발행 → 결제 → 세금계산서 → COMPLETED        │
└──────────────────────────────┬──────────────────────────────────────┘
                               │ (선택적)
┌──────────────────────────────▼──────────────────────────────────────┐
│  STAGE 11. VOC · 클레임 (화주 / 관리자)                              │
│  VOC 등록 → 클레임 접수 → 처리 → 인시던트 비용 처리                   │
└─────────────────────────────────────────────────────────────────────┘
```

|:---|:---|

| **CARRIER** (운송사) | 운임 요율 등록·관리, 배정 오더 조회 |
| **CUSTOMS_BROKER** (통관사) | 통관 요율 등록·관리, 배정 오더 통관 처리 |
| **DELIVERY_AGENT** (배송사) | 배송 요율 등록·관리, 배정 오더 배송 처리 |
| **OPERATOR** (창고 운영자) | 창고 입·출고 스캔, 재고 관리, 트래킹 |
| **MANAGER** (플랫폼 운영자) | 오더 전체 조회·관리, 경로 배정, 정산, 통관 처리 |
| **ADMIN** (시스템 관리자) | 전체 기능 + 회원 승인, 시스템 설정, 오류 로그 |

## 5. 오더 상태 머신 (Order Status Machine)

```
REGISTERED
    │ 경로 선택 완료 (MANAGER)
    ▼
SCHEDULED
    │ 창고 입고 (OPERATOR)
    ▼
WAREHOUSED
    │ 마스터 포장 (OPERATOR)
    ▼
PACKED
    │ 출고 스캔 (OPERATOR)
    ▼
RELEASED
    │ 운송 중 (시스템 자동)
    ▼
IN_TRANSIT
    │ 배송 완료 (DELIVERY_AGENT)
    ▼
DELIVERED
    │ 정산 완료 (MANAGER)
    ▼
COMPLETED

예외 흐름:
모든 상태 → HELD(보류) → 원상복구
REGISTERED / WAREHOUSED → RETURNED → DISPOSED
IN_TRANSIT / DELIVERED → CLAIMED
```

## 6. 요율 재표시 이유 (STAGE 3 vs STAGE 2)

| 구분 | 발생 시점 | 성격 | 근거 |
|:---|:---|:---|:---|
| **견적 요금** (STAGE 2) | 오더 등록 시 | 비확정 | 화주 신고 중량/CBM + 요율표 |
| **확정 요금** (STAGE 10) | 정산 시 | 확정·청구 근거 | 실측 중량/CBM + 연료할증료 + 당시 환율 |

스케줄 매핑(STAGE 3) 시점에 요금이 표시되는 이유:
- **운항편 확정 시점 연료할증료(Fuel Surcharge)** 적용 (매월 변동)
- **실측 중량 재산정**: 실제 화물 계측 후 부피중량(CBM × 167)과 비교
- **인보이스 법적 근거 확립**: 이 시점의 요금이 청구서 기준
- **화주 사전 안내**: 견적과 실제 차이를 배송 전에 투명하게 공지
