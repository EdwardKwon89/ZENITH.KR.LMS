# 다. 코드 정의 (An_03_Code_Definition)

> **프로젝트:** ZENITH_LMS (SNTL 통합 물류 플랫폼)
> **문서번호:** An-03
> **작성자:** Antigravity (AI Agent)
> **작성일:** 2026-04-16
> **버전:** v1.0

본 문서는 SNTL 통합 물류 플랫폼(ZENITH_LMS)에서 사용하는 시스템 공통 코드 및 정책 코드를 정의합니다. 모든 코드는 그룹별 챕터 구조로 기술되며, 각 코드값의 비즈니스적 의미를 포함합니다.

## 1. 회원 관련 코드 그룹

회원 계정의 분류, 상태 및 등급 정책에 사용되는 코드입니다.

### 1.1 회원 등급 코드 (GRADE)
개인회원의 이용 실적에 따라 부여되는 서비스 등급 및 할인 정책입니다.
#### 1.1.1 Family (0%) - 기본 등급
- 설명: 최초 가입 시 부여되는 기본 등급입니다. 추가 할인 혜택이 없습니다.
#### 1.1.2 Bronze (3%) - 브론즈 등급
- 설명: 기초 이용 실적을 충족한 회원에게 부여됩니다.
#### 1.1.3 Silver (5%) - 실버 등급
- 설명: 정기적인 운송 실적이 있는 회원에게 부여됩니다.
#### 1.1.4 Gold (8%) - 골드 등급
- 설명: 우수한 이용 실적을 보유한 회원에게 부여됩니다.
#### 1.1.5 Platinum (10%) - 플래티넘 등급
- 설명: 최상위 이용 실적을 보유한 회원에게 부여되는 최고 등급입니다.

### 1.2 회원 상태 코드 (MEMBER_STATUS)
계정의 활성 및 이용 제한 상태를 나타냅니다.
#### 1.2.1 ACTIVE - 활성
- 설명: 시스템의 모든 기능을 정상적으로 이용 가능한 상태입니다.
#### 1.2.2 INACTIVE - 휴면/대기
- 설명: 법인회원의 경우 가입 후 관리자 심사 대기 중인 상태를 포함합니다.
#### 1.2.3 WITHDRAWN - 탈퇴
- 설명: 회원이 직접 탈퇴하거나 관리자에 의해 삭제된 상태입니다. (Soft Delete 적용)

## 2. 오더 및 운송 관련 코드 그룹

물류 프로세스의 흐름과 서비스 유형을 정의하는 코드입니다.

### 2.1 오더 진행 상태 코드 (ORDER_STATUS)
오더의 생애주기를 관리하는 핵심 상태 코드입니다.
#### 2.2.1 REGISTERED - 접수
- 설명: 회원이 최초로 오더를 등록한 초기 상태입니다. 정보 수정이 자유롭습니다.
#### 2.2.2 PACKED - 패킹완료 (Packing)
- 설명: 마스터오더에 포함되어 묶음 처리가 완료된 상태입니다. 이때부터 하우스 오더의 **단독 수정/삭제가 제한**됩니다.
#### 2.2.3 WAREHOUSED - 입고
- 설명: 화물이 창고에 도착하여 실물 확인이 완료된 상태입니다. 이 단계 이후에는 **모든 정보의 수정 및 삭제가 원천 금지**됩니다.
#### 2.2.4 RELEASED - 출고
- 설명: 운송장이 부착되어 창고에서 출발한 상태입니다.
#### 2.2.5 DELIVERED - 배송완료
- 설명: 수하인에게 화물 인도가 완료되어 트래킹이 종료된 상태입니다.

### 2.2 운송 서비스 구분 코드 (SERVICE_TYPE)
오더에 추가되는 서비스의 유형을 구분합니다.
#### 2.2.1 AIR - 항공운송 (항운)
- 설명: 항공기를 통한 국제 운송 서비스입니다.
#### 2.2.2 SEA - 해상운송 (해운)
- 설명: 선박을 통한 국제 운송 서비스입니다.
#### 2.2.3 CIR - 국제택배 (Courier)
- 설명: 문전 연결(Door-to-Door) 기반의 국제 소형 화물 운송 서비스입니다.
#### 2.2.4 CCL - 통관 (Customs Clearance)
- 설명: 국가별 수출입 통관 대행 서비스입니다.

## 3. 재무 및 청구 관련 코드 그룹

### 3.1 청구서 상태 코드 (INVOICE_STATUS)
#### 3.1.1 UNPAID - 미납
- 설명: 청구서가 발행되었으나 입금이 확인되지 않은 상태입니다.
#### 3.1.2 PARTIAL - 부분입금
- 설명: 청구 총액 중 일부만 입금된 상태입니다.
#### 3.1.3 PAID - 완납
- 설명: 청구 총액 전액이 입금 완료된 실적 확정 상태입니다.

## 4. 데이터 표준화 및 매핑 정책 [신규]

이기종 시스템(대외 기관)으로부터 수집되는 상이한 코드 체계를 통합 관리하기 위한 정책입니다.

### 4.1 Internal Standard (내부 표준) 원칙
- 모든 비즈니스 로직과 DB 저장의 기준은 **협회/국제 표준(예: ISO 3166-1, IATA)**을 따르는 **'내부 표준 코드'**를 사용합니다.

### 4.2 외부 코드 매핑 (Standardization Mapping)
- 동일한 엔티티에 대해 원천 시스템(MOFA, MOF, Airlines 등)별로 코드가 상이한 경우, `standard_code_mapping` 테이블을 통해 내부 표준 코드로 변환하여 사용합니다.
- **예시**: 
  - 내부 표준(ISO): `KOR`
  - 매핑: `MOFA: KOR`, `Airlines: KR`, `MOF: KR` 등

### 4.3 데이터 마이그레이션 준칙
- 신규 데이터 적재 시, 매핑 테이블에 존재하지 않는 신규 시스템 코드는 관리자의 사전 매핑 정의 후 마이그레이션을 수행합니다.
