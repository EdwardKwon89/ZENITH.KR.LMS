# 5. 창고관리
**설명**: 물류 창고에서의 화물 입고·출고 처리 및 운송장 출력 기능.
**Phase**: Phase 2

## 5.1 입고 관리
**설명**: 화물이 창고에 도착하여 수령 확인 후 입고 처리하는 기능.
**Phase**: Phase 2

### 5.1.1 입고 처리
**상세**: 오더를 선택하여 입고 처리. 입고 완료 시 오더 상태가 WAREHOUSED로 변경. 입고 이후에는 오더 수정·삭제 불가. 입고 담당자명과 입고일시가 자동 기록됨. (비고: 1차 WEB 개발, 2차 PDA/App 지원 예정)
**권한**: 운영자/관리자
**Phase**: Phase 2

### 5.1.2 입고 조회
**상세**: 입고 처리된 오더 목록 및 이력 조회. 오더번호·날짜 기준 검색.
**권한**: 운영자/관리자
**Phase**: Phase 2

## 5.2 출고 관리
**설명**: 창고에서 화물을 출고하고 운송장을 출력·부착하는 기능.
**Phase**: Phase 2

### 5.2.1 바코드 스캔
**상세**: 웹캠 또는 스캐너로 바코드를 스캔하여 오더를 식별. 스캔된 오더번호를 기반으로 운송장 출력 및 출고 처리. (비고: 1차 WEB 스캔 방식 지원)
**권한**: 운영자/관리자
**Phase**: Phase 2

### 5.2.2 운송장 출력
**상세**: CIR(택배) 서비스 오더의 운송장을 PDF로 생성 및 출력. 마스터오더에 포함된 오더 정보를 기반으로 운송장 내용 자동 구성.
**권한**: 운영자/관리자
**Phase**: Phase 2

### 5.2.3 출고 처리
**상세**: 운송장 부착 확인 후 출고 처리. 출고 완료 시 오더 상태가 RELEASED로 변경. 출고 담당자명·운송장번호·출고일시 자동 기록.
**권한**: 운영자/관리자
**Phase**: Phase 2

### 5.2.4 출고 조회
**권한**: 운영자/관리자
**Phase**: Phase 2
