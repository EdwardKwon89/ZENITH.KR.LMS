# 가. 데이터 사전 (An_01_Data_Dictionary)

> **프로젝트:** ZENITH_LMS (SNTL 통합 물류 플랫폼)
> **문서번호:** An-01
> **작성자:** Antigravity (AI Agent)
> **작성일:** 2026-04-16
> **버전:** v1.0

본 문서는 ZENITH_LMS 구축에 필요한 초기 데이터 모델링 정보를 통합 관리합니다.
모든 상세 설명은 계층형 챕터(Chapter) 구조를 따르며, 비즈니스 로직 및 제약 사항을 포함합니다.

## 1. 회원 / 인증 모듈

사용자 계정, 법인 정보, 권한 및 등급 정책을 관리하는 마스터 데이터를 포함합니다.

### 1.1 individual_member (개인회원)
B2C 개인 사용자의 기본 정보를 관리하는 테이블입니다.
#### 1.1.1 id (고유ID)
- 물리명: id / 타입: BIGSERIAL / 제약: PK, NOT NULL
- 설명: 개인회원의 시스템 고유 식별자입니다. 시퀀스에 의해 자동 증가합니다.
#### 1.1.2 login_id (로그인ID)
- 물리명: login_id / 타입: VARCHAR(50) / 제약: UNIQUE, NOT NULL
- 설명: 로그인 시 사용하는 아이디입니다. 시스템 전체에서 중복될 수 없습니다.
#### 1.1.3 password (비밀번호)
- 물리명: password / 타입: VARCHAR(255) / 제약: NOT NULL
- 설명: BCrypt 라이브러리로 암호화된 비밀번호를 저장합니다.
#### 1.1.4 name (이름)
- 물리명: name / 타입: VARCHAR(100) / 제약: NOT NULL
- 설명: 회원의 법정 실명입니다.
#### 1.1.5 phone (휴대폰번호)
- 물리명: phone / 타입: VARCHAR(20) / 제약: NOT NULL
- 설명: 회원의 연락처입니다. 숫자만 포함하며 Varchar2(20) 표준을 따릅니다.
#### 1.1.6 address_info (주소 정보)
- 물리명: zipcode, address, address_detail
- 설명: 국가코드, 우편번호, 시/도, 시/군/구, 읍/면/동, 상세주소로 구성된 표준 주소 체계를 따릅니다.
#### 1.1.7 grade (회원등급)
- 물리명: grade / 타입: VARCHAR(20) / 기본값: 'iron'
- 설명: 서비스 이용 등급을 정의합니다. 등급은 iron, bronze, silver, gold로 구분되며 최초 가입 시 시스템에 의해 iron 등급이 부여됩니다.

### 1.2 corporate (법인)
B2B 법인 고객의 기본 정보를 관리합니다.
#### 1.2.1 id (고유ID)
- 물리명: id / 타입: BIGSERIAL / 제약: PK, NOT NULL
- 설명: 법인의 고유 식별자입니다. 시스템 시리얼 번호 6자리를 부여하는 규칙을 따릅니다.
#### 1.2.2 business_number (사업자등록번호)
- 물리명: business_number / 타입: VARCHAR(20) / 제약: UNIQUE, NOT NULL
- 설명: 법인의 사업자등록번호입니다. 가입 심사 시 중복 여부를 체크합니다.
#### 1.2.3 approval_status (심사상태)
- 물리명: approval_status / 타입: VARCHAR(20) / 기본값: 'PENDING'
- 설명: 관리자의 가입 승인 상태를 관리합니다. (PENDING: 대기, APPROVED: 승인, REJECTED: 거부)
#### 1.2.4 attachment_path (증빙서류 파일 경로)
- 물리명: attachment_path / 타입: TEXT
- 설명: 법인 가입 시 업로드된 사업자등록증 등 증빙 서류의 스토리지(MinIO) 저장 경로입니다.

### 1.3 orders (오더)
물류 운송 의뢰의 기본 정보를 저장하는 핵심 테이블입니다.
#### 1.3.1 order_no (오더번호)
- 물리명: order_no / 타입: VARCHAR(30) / 제약: UNIQUE, NOT NULL
- 설명: 시스템이 자동으로 생성하는 고유 오더번호입니다. (예: ORD-YYYYMMDD-SERIAL)
#### 1.3.2 order_status (오더상태)
- 물리명: order_status / 타입: VARCHAR(30) / 기본값: 'REGISTERED'
- 설명: 오더의 현재 진행 단계를 나타냅니다. REGISTERED(접수), PACKED(패킹완료), WAREHOUSED(입고), RELEASED(출고), DELIVERED(배송완료) 순으로 진행됩니다. 
- 비고: PACKED 상태 이후 혹은 "입고" 처리 이후에는 오더 정보의 수정 및 삭제가 제한됩니다.

### 1.4 cargo_info (화물정보)
오더에 포함된 화물의 상세 명세를 저장합니다.
#### 1.4.1 details (화물 상세)
- 물리명: cargo_name(품명), hs_code(HS코드), gross_weight(총중량), cbm(용적)
- 설명: 화물의 물리적 특성을 저장합니다. 총중량과 용적(CBM)은 서비스 운송비 자동 계산의 기초 데이터로 활용됩니다.
#### 1.4.2 special_cargo (특수화물 정보)
- 설명: 위험물여부(is_dangerous), 위험물코드(imdg_code), 냉동냉장여부(is_refrigerated), 보존온도(temperature), 고가품여부(is_high_value), 중고품여부(is_used)를 포함합니다. 
- 비고: 특수 화물 기재 사항인 IMDG 코드나 적정 보관 온도는 수동 기재가 필수입니다.

## 2. 코드 및 정책 모듈

시스템 전반에서 사용하는 공통 코드 및 운임/등급 정책을 관리합니다.

### 2.1 common_code (공통코드)
시스템의 모든 상태값과 분류 코드값을 통합 관리합니다.
#### 2.1.1 code_group (코드그룹)
- 설명: 코드의 논리적 그룹(예: 오더상태, 회원등급, 서비스구분)을 정의합니다.
#### 2.1.2 code (세부코드)
- 설명: 그룹 내의 실제 코드값과 명칭(Localized Name 포함)을 저장합니다.

### 2.2 grade_master (회원 등급 마스터) [신규]
개인 및 법인회원의 등급 체계와 혜택을 정의합니다.
#### 2.2.1 grade_code / name (등급 코드 및 명칭)
- 설명: Family(0%), Bronze(3%), Silver(5%), Gold(8%), Platinum(10%) 등급을 정의합니다.
#### 2.2.2 requirements (승급 요건)
- 설명: 해당 등급을 얻기 위한 '기간 내 최소 발송 횟수' 및 '최소 누적 금액' 기준입니다.
#### 2.2.3 discount_rate (등급별 할인율)
- 설명: 해당 등급에게 적용되는 기본 운송비 할인율(%)입니다. 설정에 따라 유연하게 변경 가능합니다.

### 2.3 system_config (시스템 설정) [신규]
시스템 전역에서 사용하는 가변 설정값을 관리합니다.
#### 2.3.1 config_key / value
- 설명: `GRADE_EVAL_PERIOD` (승급 심사 산정 기간, 예: 3개월), `BASE_PROFIT_MARGIN` (기본 영업이익률) 등의 키-값 쌍을 저장합니다.

### 2.4 standard_code_mapping (표준 코드 매핑) [신규]
이기종 시스템(외교부, 해수부, 항공사 등) 간의 상이한 코드 체계를 통합 관리합니다.
#### 2.4.1 category (카테고리)
- 설명: 매핑 대상의 종류 (예: COUNTRY, AIRPORT, SEAPORT, AIRLINE)
#### 2.4.2 internal_code (내부 표준 코드)
- 설명: SNTL 시스템 내부에서 사용하는 기준 코드 (예: ISO 3166-1 3자리 코드)
#### 2.4.3 source_system / source_code (원천 시스템 및 코드)
- 설명: 데이터 원천 명칭(예: MOFA)과 해당 시스템에서의 코드값(예: KOR)을 저장합니다.

### 2.5 grade_promotion_request (승급 심사 요청) [신규]
시스템이 추출한 승급 대상자와 관리자의 승인 이력을 관리합니다.
#### 2.5.1 member_id / target_grade
- 설명: 승급 대상 회원과 목표 등급입니다.
#### 2.5.2 stats_data (집계 데이터)
- 설명: 산정 기간 동안의 발송 횟수 및 누적 금액 등 심사 근거 데이터입니다.
#### 2.5.3 status (심사 상태)
- 설명: REQUESTED(시스템추출), APPROVED(승인), REJECTED(반려) 상태를 관리합니다.

### 2.2 transport_cost_master (운송원가마스터)
서비스, 운송사, 구간별 운송 원가 기준을 관리합니다.
#### 2.6.1 costs (원가 상세)
- 물리명: unit_cost(단위원가), weight_from/to(중량구간)
- 설명: 중량 또는 용적(CBM) 단위의 기초 원가입니다. 운송비 자동 계산 전 단계에서 참조됩니다.
#### 2.6.2 validity (유효 기간)
- 설명: 적용시작일(effective_from)과 적용종료일(effective_to)을 통해 원가 데이터의 히스토리를 관리합니다.
#### 2.6.3 profit_margin (영업이익률)
- 설명: 해당 서비스/구간에 대해 기본적으로 적용되는 영업이익률(%)입니다.

## 3. 운송 및 창고 모듈

화물의 물리적 이동과 마스터오더 구성을 관리하는 데이터를 포함합니다.

### 3.1 master_order (마스터오더)
여러 하우스 오더(House B/L)를 하나의 그룹으로 묶어 일괄 관리합니다.
#### 3.1.1 master_order_no (마스터오더번호)
- 설명: 시스템 일련번호 규칙에 따라 생성되는 고유 식별자입니다.
#### 3.1.2 connections (연결 정보)
- 설명: 개별 오더(orders) 테이블과 1:N 관계로 연결되어 Packing 상태를 관리합니다.

### 3.2 warehouse_receipt/release (입출고 이력)
창고 내 화물 유입과 유출을 기록합니다.
#### 3.2.1 check_info (확인 정보)
- 설명: 처리자(by), 처리일시(dt)를 기록하며 바코드 스캔 여부에 따른 식별 정보를 포함합니다.

## 4. 트래킹 및 통관 모듈

화물의 실시간 위치 정보와 관세청 신고 결과를 관리합니다.

### 4.1 tracking_info (운송트래킹)
외부 API(항공사/선사/택배사)로부터 수신한 운송 상태를 저장합니다.
#### 4.1.1 raw_data (원본 데이터)
- 물리명: raw_data / 타입: TEXT (JSON)
- 설명: 외부 API로부터 수신한 응답 원본 정보를 저장하여 이슈 발생 시 추적 용도로 활용합니다.

## 5. 회계 및 청구 모듈

운송 완료 후 정산 및 증빙 발행 업무를 관리합니다.

### 5.1 invoice (청구서)
회원에게 발행되는 최종 청구 명세입니다.
#### 5.1.1 payment_status (입금 상태)
- 설명: UNPAID(미납), PARTIAL(부분납), PAID(완납) 상태를 가집니다. 입금 금액(paid_amount)과 청구 총액(total_amount)을 비교하여 자동 업데이트됩니다.
#### 5.1.2 extra_charge (추가 비용)
- 물리명: extra_charge / 타입: DECIMAL(10, 1)
- 설명: 오더별로 수동 입력되는 추가 비용입니다. 소수점 1자리까지 지원합니다.

### 5.2 tax_invoice (세금계산서)
청구서 정보를 기반으로 발행되는 합계 잔액 증빙입니다.

## 6. 시스템 관리 모듈

메뉴, 권한, VOC 및 알림 시스템을 위한 관리 데이터입니다.

### 6.1 menu (메뉴)
시스템 화면의 계층 구조를 관리합니다.
#### 6.1.1 hierarchy (계층 구조)
- 설명: 상위메뉴ID(parent_id)를 통한 트리 구조 시스템을 지원합니다.

### 6.2 voc (고객문의)
오더별 고객 문의와 답변 내역을 저장합니다.
#### 6.2.1 context (문의 맥락)
- 설명: 특정 오더ID(order_id)와 연계되어 상담 이력을 체계적으로 관리합니다.
