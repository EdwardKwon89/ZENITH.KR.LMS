# 1. 로그인/인증
**설명**: 시스템 접속을 위한 인증 처리. JWT 기반 Stateless 인증 방식 사용.
**권한**: 전체
**Phase**: Phase 1

## 1.1 로그인
**설명**: 사용자가 시스템에 접속하기 위해 자격증명을 검증하는 기능.
**Phase**: Phase 1

### 1.1.1 ID/PW 로그인
**상세**: 아이디와 비밀번호를 입력하여 로그인. 성공 시 Access Token(30분)과 Refresh Token(7일)을 발급함. 비밀번호는 BCrypt로 해싱하여 검증.
**권한**: 회원/운영자/관리자
**Phase**: Phase 1

### 1.1.2 소셜 로그인
**상세**: Google·Kakao 등 OAuth2 소셜 계정으로 인증. 최초 로그인 시 회원가입 연동 처리. 개인회원 전용 기능. (비고: 소셜 플랫폼 추후 확정)
**권한**: 개인회원
**Phase**: Phase 1

## 1.2 ID/PW 찾기
**설명**: 로그인 정보를 분실했을 때 본인 인증을 통해 ID 확인 및 PW 재설정을 지원하는 기능.
**Phase**: Phase 1

### 1.2.1 ID 찾기
**상세**: 이름과 휴대폰번호를 입력하고 SMS 인증번호를 통해 본인 확인 후 로그인 ID를 화면에 표시. 법인회원은 법인명 + 사업자번호로 확인.
**권한**: 회원
**Phase**: Phase 1

### 1.2.2 PW 재설정
**상세**: 이메일 또는 SMS 인증을 통해 본인 확인 후 새 비밀번호를 설정. 새 비밀번호는 영문+숫자+특수문자 8자 이상 조건 적용.
**권한**: 회원
**Phase**: Phase 1

## 1.3 로그아웃
**설명**: 로그아웃 요청 시 서버 세션을 파기하고 보안 쿠키를 클리어하여 재사용을 차단.
**권한**: 회원/운영자/관리자
**Phase**: Phase 1

## 1.4 세션 거버넌스 (v2.1 신규)
**설명**: 관리자가 플랫폼 운영 정책에 따라 세션 유지 및 유휴 시간을 동적으로 제어.
**Phase**: Phase 2

### 1.4.1 지능형 유휴 타임아웃 (Idle Timeout)
**상세**: 사용자 활동이 없을 경우 보안 강화를 위해 세션을 강제 종료함.
- **기준 시간**: 10분 (기본값, `system_settings` 테이블을 통해 관리자가 실시간 변경 가능)
- **메커니즘**:
    - **유휴 감지**: `src/proxy.ts`에서 매 요청마다 `zen_idle_marker`를 검증.
    - **자동 연장 (Rolling)**: 사용자가 활동(페이지 이동, API 요청 등) 시 유휴 카운터를 다시 10분으로 초기화.
    - **강제 축출**: 10분간 활동이 없을 경우, 다음 요청 시 즉각 세션을 파기하고 로그인 페이지로 리다이렉트.

### 1.4.2 시스템 설정 연동 (Configuration-Driven)
**상세**: 하드코딩을 배제하고 DB의 `system_settings` 테이블과 연동하여 운영 정책을 실시간 반영함.
- **주요 파라미터**: `SESSION_IDLE_TIMEOUT_MIN`, `AUTH_REDIRECT_LOGIN`, `PLATFORM_VERSION` 등.
- **관리 주체**: PLATFORM 관리자 (운영사) 전용 설정 화면 제공.
