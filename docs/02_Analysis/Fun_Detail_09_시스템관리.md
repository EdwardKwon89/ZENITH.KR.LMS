# 9. 시스템관리
**설명**: 시스템 운영에 필요한 메뉴·코드·권한·알림·통계·데이터 백업을 관리.
**Phase**: Phase 4

## 9.1 메뉴 관리
**설명**: 시스템 화면 메뉴 구조를 등록·수정·삭제·조회. 계층형(트리) 구조 지원.
**Phase**: Phase 4

### 9.1.1 메뉴 등록
**상세**: 새 메뉴 항목 등록. 상위메뉴·메뉴명·URL·정렬순서 입력. 최상위 메뉴는 상위메뉴 없이 등록.
**권한**: 관리자
**Phase**: Phase 4

### 9.1.2 메뉴 수정
**상세**: 메뉴명·URL·정렬순서·사용여부 수정.
**권한**: 관리자
**Phase**: Phase 4

### 9.1.3 메뉴 삭제
**상세**: 메뉴 삭제(사용여부 N 처리). 하위 메뉴가 있는 경우 삭제 불가.
**권한**: 관리자
**Phase**: Phase 4

### 9.1.4 메뉴 조회
**상세**: 전체 메뉴 트리 구조 조회. 권한별 접근 가능 메뉴 필터링.
**권한**: 관리자
**Phase**: Phase 4

## 9.2 코드 관리
**설명**: 시스템 공통 코드(상태값·분류값 등)를 그룹별로 등록·관리.
**Phase**: Phase 4

### 9.2.1 코드 등록
**상세**: 코드그룹 내 새 코드를 등록. 코드값·코드명·정렬순서 입력.
**권한**: 관리자
**Phase**: Phase 4

### 9.2.2 코드 수정
**상세**: 코드명·정렬순서 수정. 코드값 변경은 시스템 영향 확인 후 처리.
**권한**: 관리자
**Phase**: Phase 4

### 9.2.3 코드 삭제
**상세**: 코드 삭제(사용여부 N 처리). 실제로 사용 중인 코드는 삭제 불가.
**권한**: 관리자
**Phase**: Phase 4

### 9.2.4 코드 조회
**상세**: 코드그룹별 전체 코드 목록 조회.
**권한**: 관리자
**Phase**: Phase 4

## 9.3 권한 관리
**상세**: 역할(Role)을 정의하고 메뉴별 접근 권한을 설정. 역할 예: ADMIN·OPERATOR·INDIVIDUAL·CORPORATE·DEPT_MANAGER.
**권한**: 관리자
**Phase**: Phase 4

## 9.4 택배사 관리
**상세**: 서비스에서 사용할 택배사 정보를 등록·수정·삭제·조회. 택배사코드·택배사명(한글/영문)·서비스국가·Tracking URL 관리. (비고: 기초코드로 Phase 1 단계에 포함하여 개발)
**권한**: 관리자
**Phase**: Phase 1

## 9.5 알림 관리
**설명**: 회원 대상 알림(공지·운송상태 변경 등)을 생성·발송·관리.
**Phase**: Phase 4

### 9.5.1 알림 등록
**상세**: 알림 제목·내용·발송 대상(전체/개인/법인) 설정. 발송 예약 시각 설정 가능.
**권한**: 관리자
**Phase**: Phase 4

### 9.5.2 알림 삭제
**상세**: 발송 전 미발송 알림 삭제. 발송 완료된 알림은 이력으로 보존.
**권한**: 관리자
**Phase**: Phase 4

### 9.5.3 알림 발송
**상세**: 등록된 알림을 대상 회원에게 즉시 또는 예약 발송. (비고: 발송 채널은 앱 푸시·이메일 중 추후 확정. 발송 결과SENT/FAILED 기록)
**권한**: 관리자
**Phase**: Phase 4

### 9.5.4 알림 조회
**상세**: 알림 발송 이력 조회. 발송상태·발송일시·대상 정보 확인.
**권한**: 관리자
**Phase**: Phase 4

## 9.6 통계 관리
**설명**: 운송·비용 데이터를 집계하여 경영 현황을 파악하는 통계 기능.
**Phase**: Phase 4

### 9.6.1 운송 통계
**상세**: 기간별·운송수단별·회원별 오더 건수·중량·CBM을 집계하여 운송 현황 통계 제공.
**권한**: 운영자/관리자
**Phase**: Phase 4

### 9.6.2 비용 통계
**상세**: 기간별 운송비 수입·원가·이익을 집계하여 수익성 분석. (비고: 세부 통계 항목은 추후 확정)
**권한**: 운영자/관리자
**Phase**: Phase 4
