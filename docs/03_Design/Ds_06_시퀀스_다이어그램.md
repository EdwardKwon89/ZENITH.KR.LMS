## 1. 오더 등록 및 계약 기반 운임 산정 (Order & Estimation)
화주가 오더를 등록할 때, 등록된 계약(Contract) 및 요율(Rate Card) 정보를 바탕으로 예상 운임을 즉시 산출합니다.

```mermaid
sequenceDiagram
    autonumber
    participant S as Shipper (User)
    participant UI as Form (Frontend)
    participant SA as Server Action
    participant DB as Supabase DB

    S->>UI: 오더 등록 페이지 진입
    UI->>SA: getSessionContext() 요청
    SA->>DB: 사용자 org_id 조회
    DB-->>SA: org_id 반환
    SA-->>UI: 사용자 타입 (B2B/B2C) 반환

    alt B2B (org_id 존재)
        UI->>UI: Shipper를 소속 법인으로 고정 (Read-only)
    else B2C (org_id 미존재)
        UI->>UI: 시스템 더미 화주 자동 할당 
    end

    S->>UI: 화물 정보(Dimension, Weight) 입력
    UI->>UI: 실시간 Chargeable Weight 계산
    Note right of UI: Air: L*W*H/6000 <br/> Sea: R.T (1000kg/1CBM)
    UI->>UI: Indigo 요약 카드 노출

    S->>UI: '오더 등록' 클릭
    UI->>SA: createOrder(data) 호출
    SA->>SA: validateUserAction(org_id) 검증
    SA->>DB: 오더 데이터 저장
    DB-->>SA: Success (Order ID)
    SA-->>UI: 등록 완료 및 결과 화면 이동
```

## 2. 운항 스케줄 사후 배정 (Post-Schedule Assignment)
등록된 오더에 대해 운영자가 실제 운송 수단(항공/해상/육상)의 스케줄을 배정합니다.

```mermaid
sequenceDiagram
    autonumber
    participant O as Operator (Admin)
    participant P as Platform (Backend)
    participant DB as Supabase DB
    participant C as Carrier (System)

    O->>P: 미배정 오더 목록 조회
    P->>DB: Status: 'REGISTERED' 오더 검색
    DB-->>P: 오더 목록 반환
    O->>P: 가용 운항 스케줄(Schedules) 검색
    P->>DB: Transport Schedules 조회 (Route + Date)
    DB-->>P: Schedules 목록 반환
    O->>P: 특정 오더에 스케줄 배정 (Assign)
    P->>DB: Order 업데이트 (schedule_id, Status: 'SCHEDULED')
    DB-->>P: 처리 완료
    P->>C: 운송사 배차 알림 (Webhook/Push)
    P-->>O: 배정 완료 결과 노출
```

## 3. 마스터 오더 생성 및 SSCC-18 식별자 발급 (Master Order)
운영자가 여러 하우스 오더를 통합하여 마스터 오더를 생성하고 글로벌 표준 식별자(SSCC-18)를 부여하는 프로세스입니다.

```mermaid
sequenceDiagram
    autonumber
    participant O as Operator (Admin)
    participant M as Master Engine
    participant L as Luhn Alg
    participant DB as Supabase DB

    O->>M: 마스터 오더 생성 요청 (New Master)
    M->>M: Extension Digit(1) + Prefix(88012345) 조합
    M->>DB: 최신 Serial Reference 조회
    DB-->>M: Serial (ex: 00000001)
    M->>L: 17자리 데이터 전달 (AI 제외)
    L->>L: Modulo 10 연산 수행
    L-->>M: Check Digit 반환 (ex: 2)
    M->>M: 최종 SSCC-18 번호 합성
    Note left of M: (00) 1 88012345 00000001 2
    M->>DB: 마스터 오더 저장 (Status: 'CREATED')
    DB-->>O: 마스터 오더 생성 완료 및 라벨 출력 준비
```

## 4. 항공 물류 특화: 부피 중량 및 구간 요율 로직
항공 물류의 경우, 다음의 복합 로직이 시스템 내부적으로 수행됩니다.

1.  **Volumetric Weight**: `(Width * Height * Length) / Divisor` (기본 6,000)
2.  **Chargeable Weight**: `MAX(Actual Weight, Volumetric Weight)`
3.  **Tier Pricing**:
    - Weight < 45kg: Min Rate 적용
    - 45kg <= Weight < 100kg: +45 Tier Rate 적용
    - Weight >= 100kg: +100 Tier Rate 적용
