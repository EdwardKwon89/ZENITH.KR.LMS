> **작성자:** Antigravity (AI Agent)
> **작성일:** 2026-04-16
> **버전:** v1.0

본 문서는 오더 등록, 마스터오더 구성 및 창고 운영 관리를 위한 상세 화면 설계를 정의합니다.

## 1. 하우스 오더(Hotel Order) 관리 화면

### 1.1 [SCR-021] 오더 신규 등록 화면
사용자가 운송 의뢰를 생성하는 시작점입니다.
#### 1.1.1 입력 영역 구성
- **주소 정보**: 송하인(Shipper) 및 수하인(Consignee)의 국가, 우편번호, 상세 주소를 입력합니다.
- **화물 명세**: 품명, HS코드, 수량(PCS/PKGS), 총중량(KG), 용적(CBM)을 입력합니다.
- **특수 화물 옵션**: 위험물, 냉동냉장 여부 선택 시 IMDG 코드 및 온도 입력 팝업을 호출합니다.
#### 1.1.2 운송비 조회 및 확정 액션
- 입력된 중량/용적을 바탕으로 **운송비 자동 계산 엔진**을 호출하여 예상 운임을 노출합니다.
- **결과**: "오더 등록" 버튼 클릭 시 `order_no`가 발급되며 'REGISTERED' 상태로 저장됩니다.

### 1.2 [SCR-020] 오더 목록 및 상세 화면
#### 1.2.1 조회 필터 및 목록
- 기간, 상태(REGISTERED~DELIVERED), 오더번호, 송/수하인명으로 검색 가능합니다.
#### 1.2.2 정보 수정 제약 (Locking Policy)
- **수정 버튼**: 오더 상세화면에서 'REGISTERED' 상태일 때만 활성화됩니다.
- **제약 사항**: 해당 오더가 마스터오더에 포함(PACKED)되거나 창고에 입고(WAREHOUSED)된 이후에는 **수정/삭제 버튼이 비활성화**되며 읽기 전용으로 전환됩니다.

## 2. 마스터 오더 및 패킹 화면

### 2.1 [SCR-031] 오더 패킹(Packing) 화면
개별 오더들을 모아 마스터오더(B/L)를 구성합니다.
#### 2.1.1 대상 선택 그리드
- 'REGISTERED' 상태의 오더 목록 중 목적국가가 동일한 오더들을 선택(Checkbox)합니다.
#### 2.1.2 패킹 확정 액션
- 'Packing 확정' 버튼 클릭 시 마스터오더ID가 생성되며, 포함된 모든 하우스 오더의 상태를 **'PACKED'**로 일괄 변경합니다.

## 3. 창고 입출고 운영 화면

### 3.1 [SCR-040] 입고 처리 화면 (Warehouse Receipt)
창고 도착 화물의 실물을 확인하고 전산 입고를 수행합니다.
#### 3.1.1 입고 확인 단계
- 오더번호 바코드 스캔 시 해당 화물의 정보를 팝업합니다.
- **확정 로직**: '입고 확정' 클릭 시 상태가 **'WAREHOUSED'**로 변경됩니다. (이후 수정 원천 불가)

### 3.2 [SCR-041] 출고 및 운송장 관리 화면
#### 3.2.1 라벨링 및 출고
- 국제택배(CIR) 오더의 경우 시스템에서 생성된 PDF 운송장(Waybill)을 출력합니다.
- **출고 액션**: 운송장 부착 후 '출고 확정' 처리 시 상태가 **'RELEASED'**로 변경됩니다.
