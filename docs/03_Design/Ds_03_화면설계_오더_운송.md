본 문서는 오더 등록, 마스터오더 구성 및 창고 운영 관리를 위한 상세 화면 설계를 정의합니다.

## 1. 하우스 오더(Hotel Order) 관리 화면

### 1.1 [SCR-021] 오더 신규 등록 화면
사용자가 운송 의뢰를 생성하는 시작점입니다.
#### 1.1.1 입력 영역 구성
- **운송 모드 선택**: AIR, SEA, LAND/CIR 중 선택 (기본값: AIR).
- **주소 정보**: 송하인(Shipper) 및 수하인(Consignee)의 국가, 우편번호, 상세 주소를 입력합니다.
- **화물 명세**: 품명, HS코드, 수량(PCS/PKGS), 중량(KG), 가로/세로/높이(CM)를 입력합니다.
- **특수 화물 옵션**: 위험물, 냉동냉장 여부 선택 시 IMDG 코드 및 온도 입력 팝업을 호출합니다.
#### 1.1.2 운송비 조회 및 확정 액션
- **예상 운임 자동 계산**: 
    - 입력된 정보와 화주별 **계약(Contract)** 정보를 실시간 매칭합니다.
    - 항공의 경우 부피 중량(Volumetric Weight)을 계산하여 **Chargeable Weight**를 노출합니다.
    - 구간별 요율(Weight Break)이 적용된 예상 운임을 화면 하단에 표시합니다.
- **결과**: "오더 등록" 버튼 클릭 시 `order_no`가 발급되며 'REGISTERED' 상태로 저장됩니다.

### 1.2 [SCR-020] 오더 목록 및 상세 화면
#### 1.2.1 조회 필터 및 목록
- 기간, 상태(REGISTERED~DELIVERED), 오더번호, 송/수하인명으로 검색 가능합니다.
#### 1.2.2 정보 수정 및 배정 정보
- **배차/운항 정보**: 오더가 마스터오더에 포함되거나 스케줄이 배정된 경우, 배정된 **항공편명/선박명/차량번호**를 노출합니다.
- **제약 사항**: 해당 오더가 마스터오더에 포함(PACKED)되거나 창고에 입고(WAREHOUSED)된 이후에는 **수정/삭제 버튼이 비활성화**됩니다.

## 2. 운영 관리 및 스케줄 배정

### 2.1 [SCR-030] 운송 스케줄 관리 화면
운영자가 실제 운송 수단의 일정을 등록하고 관리합니다. (AIR/SEA/LAND 공용)
- **스케줄 등록**: 운송사 선택, 편명(Vessel/Flight No), 출발/도착지 및 ETD/ETA 입력.

### 2.2 [SCR-031] 오더-스케줄 배정(Assignment) 화면
'REGISTERED' 상태의 오더들을 실제 운항 스케줄에 할당합니다.
- **대상 선택**: 미배정 오더 목록에서 목적지/품목이 동일한 오더들을 선택.
- **배정 액션**: '스케줄 확정' 클릭 시 해당 오더들의 상태를 **'SCHEDULED'**로 변경하고 운항 정보를 연결합니다.

## 3. 창고 입출고 운영 화면

### 3.1 [SCR-040] 입고 처리 화면 (Warehouse Receipt)
창고 도착 화물의 실물을 확인하고 전산 입고를 수행합니다.
#### 3.1.1 입고 확인 단계
- 오더번호 바코드 스캔 시 해당 화물의 정보를 팝업합니다.
- **확정 로직**: '입고 확정' 클릭 시 상태가 **'WAREHOUSED'**로 변경됩니다. (이후 수정 원천 불가)

### 3.2 [SCR-041] 출고 및 운송장 관리 화면
#### 3.2.1 라벨링 및 출고
- 국제택배(CIR) 오더의 경우 시스템에서 생성된 PDF 운송장(Waybill)을 출력합니다.
- **출고 액션**: 운송장 부착 후 '출고 확정' 처리 시 상태가 **'RELEASED'**로 변경됩니다.
