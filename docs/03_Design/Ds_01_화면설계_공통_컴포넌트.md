# 마. 화면 설계 - 공통 컴포넌트 (Ds_01_Common_UI)

> **문서번호:** Ds-01
> **작성자:** Antigravity (AI Agent)
> **작성일:** 2026-04-16
> **버전:** v1.0

본 문서는 ZENITH_LMS 시스템의 전반적인 레이아웃 프레임워크와 재사용 가능한 공통 UI 컴포넌트를 정의합니다.

## 1. 시스템 레이아웃 프레임워크

### 1.1 전체 구조 (Main Layout)
시스템은 반응형 웹 구조를 기본으로 하며, 다음과 같은 3가지 주요 영역으로 구성됩니다.
#### 1.1.1 상단 내비게이션 (GNB)
- **구성**: 서비스 로고, 통합 검색 바, 알림 아이콘, 내 정보/미니 프로필
- **동작**: 스크롤 시 상단에 고정(Sticky)되며, 고유 ID는 `gnb-main`을 사용합니다.
#### 1.1.2 측면 메뉴바 (LNB)
- **구성**: 시스템 메뉴 트리 (메인 메뉴 - 서브 메뉴), 접기/펴기 버튼
- **구조**: 메뉴 권한 테이블(`menu_role`)에 정의된 권한에 따라 노출 여부가 동적으로 결정됩니다.
#### 1.1.3 메인 본문 영역 (Content Area)
- **구성**: 페이지 타이틀, 브레드크럼(Breadcrumb), 실제 콘텐츠 뷰
- **규칙**: 최소 너비 1280px를 기준으로 설계하며 그리드 시스템을 적용합니다.

## 2. 공통 UI 컴포넌트 명세

### 2.1 버튼 시스템 (Buttons)
가시성과 중요도에 따른 4단계 버튼 스타일을 정의합니다.
#### 2.1.1 Primary Button (주요 액션)
- 사용: 등록, 수정, 승인 등 핵심 액션
- 스타일: 시스템 메인 컬러 배경, 화이트 텍스트
#### 2.1.2 Secondary Button (보조 액션)
- 사용: 취소, 이전, 닫기 등 보조 액션
- 스타일: 그레이 아웃라인 또는 라이트 그레이 배경
#### 2.1.3 Action Button (목록 액션)
- 사용: 엑셀 다운로드, 인쇄, 조회 등 목록 기반 액션
- 스타일: 아이콘 결합형 스타일

### 2.2 입력 및 폼 요소 (Inputs & Forms)
#### 2.2.1 데이터 입력 프레임워크
- **공통**: 모든 필수 입력 필드는 붉은색 애스터리스크(*)를 명시합니다.
- **포커스 유효성**: Focus-out 시 실시간 데이터 포맷 검증(Validation)을 수행합니다.
#### 2.2.2 날짜 선택기 (Date Picker)
- 구성: YYYY-MM-DD 형식의 Input과 캘린더 팝업UI
- 규칙: 출발/도착일 선택 시 시작일이 종료일보다 늦을 수 없는 로직을 포함합니다.
#### 2.2.3 코드 선택박스 (Select Box)
- 설명: `An_03_Code_Definition`에 정의된 공통 코드 그룹 데이터를 비동기(Axios)로 호출하여 목록을 구성합니다.

## 3. 공통 알림 및 피드백UI

### 3.1 모달 및 알럿 (Modals & Alerts)
#### 3.1.1 안내 및 경고 (Alerts)
- 형태: 페이지 상단에 일시적으로 노출되는 토스트(Toast) 메시지
#### 3.1.2 확인 모달 (Confirm Modals)
- 형태: 중앙 레이어 팝업으로 사용자에게 최종 결정을 묻는 UI (예: "정말 삭제하시겠습니까?")
