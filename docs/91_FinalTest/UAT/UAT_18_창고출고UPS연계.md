## [UAT-18-01] WAREHOUSED 오더 출고 완료 시 UPS 발송 연계 흐름

| 항목 | 내용 |
|:----|:----|
| 역할 | MANAGER (창고 운영자) 또는 ADMIN |
| 화면 URL | /ko/warehouse/outbound |
| 예상 소요 시간 | 5분 |
| 사전 조건 | 입고 완료(`WAREHOUSED`) 상태이며 운송 모드가 `UPS Express`인 오더가 1건 이상 존재할 것 |

### 테스트 절차

| 순서 | 화면·URL | 수행 액션 | 입력 데이터 | 기대 결과 | 확인 |
|:---:|:---------|:---------|:-----------|:---------|:----:|
| 1 | /ko/warehouse/outbound | 사이드바 > 창고 관리 > [출고 처리] 메뉴 클릭 | — | 출고 대기 목록 화면 진입 | ☑ |
| 2 | /ko/warehouse/outbound | 대상 UPS 오더 선택 후 [출고 확정] 또는 [배송 처리] 버튼 클릭 | — | 출고 상세/컨펌 화면 표시 | ☑ |
| 3 | /ko/warehouse/outbound | 출고 정보 검수 후 최종 [출고 완료] 클릭 | — | "출고 처리가 완료되었으며 UPS 송장이 채번되었습니다." 알림 표시 및 오더 상태가 `RELEASED`로 업데이트됨 | ☑ |
| 4 | /ko/warehouse/outbound | 목록에서 상태 및 송장 번호 확인 | — | 출고 이력 테이블에서 해당 오더의 상태가 `출고완료(RELEASED)`로 표시되고, UPS 해외 송장 번호(Tracking No)가 연동되어 노출됨 | ☑ |
| 5 | Supabase Studio | `SELECT status, intl_ref_no FROM zen_orders WHERE id = '[오더ID]'` | — | `status = 'RELEASED'` 및 `intl_ref_no` (UPS 해외송장번호) 정상 적재 확인 | ☑ |

### 합격 기준
- [x] 전 단계 ☑ 완료
- [x] UPS 오더 출고 시 백엔드에서 가상의 UPS API(또는 RPC)가 성공적으로 연동되어 `intl_ref_no`가 자동 입력됨
- [x] 오더의 상태가 `RELEASED`로 문제없이 전이됨
- [x] 500 에러 없음

## [UAT-18-02] UPS 발송 정보 자동 매핑 및 RLS 격리 검증

| 항목 | 내용 |
|:----|:----|
| 역할 | SHIPPER (화주) |
| 화면 URL | /ko/orders |
| 예상 소요 시간 | 5분 |
| 사전 조건 | UAT-18-01 과정을 통해 출고 완료 및 UPS 송장이 채번된 오더가 존재할 것 |

### 테스트 절차

| 순서 | 화면·URL | 수행 액션 | 입력 데이터 | 기대 결과 | 확인 |
|:---:|:---------|:---------|:-----------|:---------|:----:|
| 1 | /ko/orders | 해당 오더를 소유한 화주 계정으로 로그인 후 [오더 목록] 페이지 조회 | — | 오더 목록에 해당 오더 표시됨 | ☑ |
| 2 | /ko/orders | 오더 상세 정보 확인 | — | 상세 카드에서 출고 시 자동 적재된 `해외 송장 번호(intl_ref_no)`를 정상적으로 볼 수 있음 | ☑ |
| 3 | /ko/orders | 다른 화주 계정으로 로그인하여 동일 오더 상세 URL 직접 접근 시도 | — | 접근 권한 오류 또는 리다이렉트 발생 (타 화주의 UPS 발송 상세 데이터 접근 불가 검증) | ☑ |

### 합격 기준
- [x] 전 단계 ☑ 완료
- [x] 출고 처리로 채번된 해외 송장 번호 정보가 화주의 오더 상세 화면에 정확히 표시됨
- [x] RLS 격리가 작동하여 다른 화주는 타인 오더의 UPS 발송 정보를 조회할 수 없음
- [x] 500 에러 없음
