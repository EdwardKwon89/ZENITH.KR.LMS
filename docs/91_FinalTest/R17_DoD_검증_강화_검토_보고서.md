---

## 1. 문제 정의

TASK 완료 후 문서 커밋 시 DoD 체크리스트의 증거값(커밋 해시·파일 경로·상태)이 `기재 예정` 또는 빈 값으로 방치됨. 코드 구현과 테스트 통과에 집중한 나머지 마지막 검증 단계를 생략하는 패턴이 반복됨.

**근본 원인**: 문서 커밋 직전 task file의 DoD를 재확인하는 행위가 강제되지 않음. 커밋 명령과 DoD 검증 사이에 물리적 장치(rule)가 없음.

---

## 2. 제안

### Option A: R-17 §6 신설 (GOV_COMMON.md)

```
§6. 문서 커밋 전 DoD 실물 검증
문서 커밋(`git commit`) 직전, 반드시 task file의 DoD 섹션을 Read하여 
각 항목의 증거값(해시·파일 경로·상태)이 실제 기재되었는지 눈으로 확인한다.
`기재 예정`, `TBD`, 빈 값이 발견되면 커밋을 보류하고 즉시 보완한다.
위반 시 R-17 §5와 동일한 제재(반려 · 경고 · 신규 할당 중단)를 적용한다.
```

| 측면 | 평가 |
|:---|:---|
| 적용 범위 | 전 Agent (Riley·B_Kai·N_Kai·Ring 포함) |
| 제재 근거 | 명확 — 기존 R-17 제재 체계와 연동 |
| 단점 | R-17 규정 추가로 인한 복잡성 증가 |

### Option B: AGENTS.md D_Kai 절차 명시

```
### 문서 커밋 전 DoD 검증 (D_Kai 전용)
문서 커밋(`git commit`) 직전:
1. `read .agent/tasks/TASK-{ID}_*.md` 실행
2. DoD 섹션의 증거값(해시·상태) 전항목 눈확인
3. 미기재 발견 시 커밋 보류 → 즉시 보완
```

| 측면 | 평가 |
|:---|:---|
| 적용 범위 | D_Kai 단독 |
| 제재 근거 | 약함 — AGENTS.md는 규정 아닌 역할 정의 |
| 장점 | 경량 · 타 Agent 영향 없음 |

## 3. 권장

**Option A + Option B 병행**을 권장합니다.

- **GOV_COMMON.md R-17 §6**: 전 Agent 공통 규칙으로 격상 — 반복 위반 Agent에 대한 제재 근거 제공
- **AGENTS.md D_Kai 절차**: 내 개인 워크플로에 구체적 실행 단계로 명시 — "규정이 내 행동에 닿는 거리"를 최소화

규정만으로 행동이 바뀌지 않는다는 것을 이번 위반으로 확인했습니다. AGENTS.md에 제 커밋 직전 수행해야 할 구체적 명령어(`read .agent/tasks/TASK-{ID}_*.md`)까지 명시하여, 규정이 실제 터미널 동작에 직접 연결되도록 해야 합니다.
