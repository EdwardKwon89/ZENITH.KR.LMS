# ZENITH_LMS 운영 가이드: 운영자(Operator) 편

> **문서번호**: MAN-OPR-01
> **대상**: 물류 센터 운영자 및 실무 담당자
> **최종 업데이트**: 2026-04-30

본 문서는 ZENITH_LMS 플랫폼에서 실물 흐름(창고, 배차, 트래킹) 및 정산 실무를 담당하는 운영자를 위한 가이드입니다.

---

## 1. 창고 관리 및 재고 실무
물류 센터 입고부터 출고까지의 모든 프로세스를 바코드로 제어합니다.

### 1.1 입/출고 처리
1. 메뉴: `창고 관리 > 입출고 관리` 이동.
2. **입고**: 화물 도착 시 운송장의 바코드를 스캔하여 `IN_WAREHOUSE` 상태로 전환합니다.
3. **출고**: 최종 패킹 완료 후 배차 차량에 상차 시 바코드를 스캔하여 `OUT_FOR_DELIVERY` 상태로 전환합니다.

### 1.2 실시간 재고 대시보드
- `창고 관리 > 재고 현황`에서 SKU별 현재 수량을 확인합니다.
- **재고 조정**: 파손, 분실 등 사유 발생 시 `재고 조정` 버튼을 클릭하여 수량을 수정하고 사유를 입력합니다. (로그 자동 기록)
- **안전 재고 알림**: 설정된 수량 미만으로 하락 시 대시보드에 빨간색 경고 배지가 노출됩니다.

## 2. 운송 트래킹 및 가시성 관리
화물의 이동 상태를 실시간으로 모니터링하고 오류 발생 시 조치합니다.

- **통합 트래킹**: 오더 상세 페이지에서 화물의 현재 위치와 실시간 통관 진행 상태(SUBMITTED -> APPROVED 등)를 확인합니다.
- **Raw Log 조회**: 데이터 불일치 또는 전송 에러 의심 시 `시스템 로그 > 트래킹/통관 로그`에서 외부 시스템(어댑터)과의 송수신 원본 데이터를 검증합니다.
- **예외 처리**: 어댑터 전송 실패 시, 해당 건을 다시 `PENDING`으로 전환하거나 관리자 권한을 통해 수동으로 상태를 보정합니다.

## 3. 오더 처리 및 배차 관리
접수된 오더를 실제 운송 경로에 할당하고 관리합니다.

### 3.1 최적 경로 선택
1. 메뉴: `오더 관리 > 배차 관리` 이동.
2. 시스템이 추천하는 3종 경로(COST, TIME, BALANCED)를 확인합니다.
3. 상황에 맞는 경로를 선택하고 `경로 확정` 버튼을 클릭합니다.

### 3.2 마일스톤 관리
- 각 운송 구간(출발지 → 경유지 → 도착지)의 도착/출발 시각을 관리하여 화주에게 실시간 정보를 제공합니다.

## 4. 정산 실무 및 데이터 관리
운송 완료된 오더에 대해 세금계산서를 발행하고 관련 데이터를 추출합니다.

- **세금계산서 발행**: `정산 관리 > 인보이스 목록`에서 대상 건을 선택하고 `세금계산서 발행` 버튼을 클릭합니다. (화주 이메일로 자동 발송)
- **엑셀 데이터 Export**: 정산 내역을 기간별, 화주별로 필터링하여 엑셀(.xlsx) 파일로 내려받아 내부 회계 시스템과 대조합니다.

## 5. 클레임 및 문서 관리
운송 중 사고 발생 시 클레임을 접수하고 필수 수출입 문서를 관리합니다.

- **사고비 등록**: 파손 또는 지연 사고 발생 시 `클레임 관리` 메뉴에서 사고 비용을 등록합니다. 이 비용은 다음 인보이스 발행 시 자동 차감됩니다.
- **CI/PL 발행**: 오더 상세 페이지의 `문서 관리` 섹션에서 상업송장(CI)과 포장명세서(PL)를 PDF로 생성하여 다운로드하거나 화주에게 전송합니다. (다국어 지원)

---
**실무 팁: 바코드 스캔 실패 시**
- 브라우저의 입력 포커스가 `스캔 입력창`에 위치해 있는지 확인하십시오.
- 운송장 번호를 수동으로 입력한 후 `Enter` 키를 눌러 처리할 수 있습니다.
