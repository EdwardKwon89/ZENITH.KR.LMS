# 하우스 오더 등록 비즈니스 로직 및 UI 명세 (v1.0)

이 문서는 사용자의 소속(Affiliation)에 따른 오더 등록 폼의 동적 제어 규칙을 정의합니다.

## 1. 사용자 유형 분류 및 정책 (User Policy)

| 사용자 유형 | 판별 조건 | 허용 오더 유형 | 화주(Shipper) 보정 | UI 제약 사항 |
|:---:|:---|:---:|:---|:---|
| **법인 회원** | `org_id` ≠ null | B2B (일반 화물) | 소속 법인으로 고정 | B2C 탭 비활성화, 화주 선택기 잠금 |
| **개인 회원** | `org_id` = null | B2C (이커머스/특송) | 시스템 개인 화주 ID | B2B 탭 비활성화, 화주 선택기 잠금 |

## 2. 시나리오별 상세 인터랙션 (Scenario Spec)

### 시나리오 A: 법인(B2B) 사용자 진입
1. **초기 로드**: 'B2B 일반 화물' 탭이 기본 선택됨.
2. **탭 제어**: 
   - 'B2C 이커머스', 'B2C 특송' 탭 상단에 "법인 회원 전용 페이지입니다" 툴팁 노출 및 클릭 차단.
3. **화주 선택**: 
   - `shipper_id`에 사용자의 `org_id`를 자동으로 주입.
   - 드롭다운 컴포넌트를 `ReadOnly` 또는 `Disabled` 상태로 변경하여 수정 원천 차단.

### 시나리오 B: 개인(B2C) 사용자 진입
1. **초기 로드**: 'B2C 이커머스' 또는 'B2C 특송' 탭 중 하나가 기본 선택됨.
2. **탭 제어**: 
   - 'B2B 일반 화물' 탭을 비활성화 처리.
3. **화주 선택**: 
   - 시스템에서 미리 약속된 '개인 화주 더미 ID'를 자동으로 주입.
   - 화주 선택 필드를 시각적으로 고정(Fixed) 처리.

## 3. 데이터 무결성 가드 (Data Integrity)

### 서버 측 검증 (Server-side Validation)
- 클라이언트 측의 UI 제약은 편의성을 위함이며, 실제 저장은 서버 액션에서 다음을 재검증함:
  - 사용자가 법인 멤버인데 B2C 오더를 전송했는가? → **거부 (403)**
  - 사용자가 제출한 `shipper_id`가 자신의 `org_id`와 일치하는가? → **거부 (403)**
  - 개인 사용자인데 B2B 오더를 전송했는가? → **거부 (403)**

## 4. 예외 사항 (Exceptions)
- **ZENITH_SUPER_ADMIN**: 플랫폼 운영자는 모든 탭에 접근 가능하며, 모든 화주를 선택할 수 있는 권한을 유지함.
