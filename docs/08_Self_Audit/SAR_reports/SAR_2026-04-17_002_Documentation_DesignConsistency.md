## 현상 (What)

비즈니스 로직(회원 등급, 산식 등)이 최종 확정되었음에도 불구하고, 기존에 작성된 일부 설계 문서에서 레거시 명칭이 잔존하거나 넘버링이 어긋나는 정합성 결함이 발견됨.

**발생 위치:**
1. `An_02_업무흐름정의.md`: 초기 등급명이 'iron'으로 기재됨 (Line 25)
2. `An_04_화면목록.md`: SCR-094 번호가 '1.3.1'로 관리자 화면 체계와 불일치 (Line 95)
3. `WBS_01_상세_공정표.md`: 등급 마스터 명칭이 'Iron~Gold'로 잔존 (Line 67)

## 원인 (Why)

### 직접적 원인
반복적인 비즈니스 로직 수정 과정에서 문구 전수 조사가 누락됨.

### 근본 원인
주요 비즈니스 정책 변경 시 관련 모든 문서의 정합성을 교차 검증하는 '문서 동기화(Cross-doc Sync)' 절차가 명시되지 않음.

### 기여 요소
- 문서가 세분화(An_01 ~ An_05)됨에 따라 개별 수정 시 전체 맥락 파악이 어려움.

## 조치 (How)

### 수정 전 (예시)
```markdown
- **비즈니스 규칙**: 최초 가입 시 회원 등급은 자동으로 'iron'이 부여되며...
```

### 수정 후
```markdown
- **비즈니스 규칙**: 최초 가입 시 회원 등급은 자동으로 'Family'가 부여되며...
```

### 수정 범위
- [x] 분석 문서(An_02, An_04) 현행화
- [x] 공정표(WBS) 마스터 데이터 명칭 수정
- [x] 시퀀스 다이어그램(An_05) 링크 및 레이블 검증

## 검증 (Verification)

### 수동 테스트
- [x] 각 문서 간 명칭(Family, Bronze, Silver, Gold, Platinum) 일관성 전수 조사 완료
- [x] 화면 번호 체계(5.x 대역) 정합성 확인

## 예방 (Prevention)

### Check List에 추가할 항목
```
□ Cross-doc Sync: 주요 비즈니스 정책 변경 시 관련 파일(An, Fun, WBS) 간 명칭/번호 정합성 확인 (SAR-002)
```

### 설계 개선
- 비즈니스 핵심 상수(예: 등급명, 산식)는 가급적 `An_03_코드정의.md`를 단일 진실 공급원(SSOT)으로 삼고 타 문서에서는 이를 적극 참조하도록 가이드 보완.
