# SAR (Self Audit Report) - E2E-01 테스트 버그 및 누락 보완

- **Issue ID**: BUG-E2E-01-01, REWORK-E2E-01-01~03
- **Date**: 2026-04-30
- **Author**: Riley (CPO)
- **Component**: E2E 브라우저 테스트 (Playwright)

## 2. 문제 현상
- **REWORK-E2E-01-01**: 관리자 페이지 URL 접근 시 `/ko/organizations`로 잘못 지정되어 404/접근 불가 현상 발생 가능.
- **REWORK-E2E-01-02**: 화주 로그인 성공 여부를 입증할 시각적 증거(스크린샷)가 누락되어 테스트 신뢰성 저하.
- **REWORK-E2E-01-03**: 화주의 무단 관리자 경로(`/ko/admin/customs`) 접근 시 인가 통제(403 또는 리다이렉트) 여부를 검증하는 로직 부재.

## 3. 원인 분석
- **REWORK-E2E-01-01**: 시나리오 명세(`docs/99_Manual/E2E_SCENARIOS.md`)와 스크립트 간 라우트 정의 불일치.
- **REWORK-E2E-01-02**: `page.screenshot` 호출 코드 누락.
- **REWORK-E2E-01-03**: 권한 제어 검증 단계(Step 8) 코드 작성 누락.

## 4. 해결 방안 (수정 내역)
- `scratch/e2e_01_verify.mjs` 스크립트 보완.
  - 관리자 조직 관리 URL을 `/ko/admin/organizations`로 올바르게 수정.
  - 로그인 성공 직후 `/ko/orders` 리다이렉트 확인 및 `e2e_01_login_success.png` 촬영.
  - 화주 세션 상태에서 `/ko/admin/customs` 경로 강제 진입 시도 후 결과 URL 검사, 인가 거부 확인 및 `e2e_01_access_control_success.png` 촬영 로직 추가.

## 5. 검증 결과
- 수정한 스크립트 실행 완료. (Exit code: 0)
- 화주가 Admin URL 진입 시 `/ko/orders` 또는 `/ko/dashboard`로 리다이렉트되는 권한 제어 정상 동작 확인.
- 모든 스크린샷 정상 생성 후 `docs/99_Manual/E2E_01_Result/`로 이동 조치 완료.

## 6. 재발 방지 대책
- 신규 E2E 테스트 스크립트 작성 시 검증 시나리오의 접근 권한 체크 단계가 코드에 정확히 1:1 매핑되었는지 `LIVE_PHASE_5_EXECUTE.md` 체크리스트 항목 점검 필수 수행.
