# SAR - 다중 경로 정산 연계 — 요구사항↔구현↔UAT 정합성 3단계 누락

**문서번호:** SAR-2026-05-23-005  
**날짜:** 2026-05-23  
**작성자:** Aiden (Claude, ZEN_CEO)  
**심각도:** HIGH (요구사항·구현·UAT 3단계 연속 누락)

## 1. 현상 (What)

- Aiden이 UAT Sprint 완료 후 다중 경로(multi-leg) 물류 배정 및 정산 시나리오가 UAT 56개에 포함되는지 사용자 질의를 통해 점검한 결과 3단계 누락을 발견함.
- REQUIREMENTS.md 3.4의 "지능형 라우팅 및 견적 추천" 항목에서 출발하는 데이터 흐름이 정산 엔진 및 UAT 시나리오까지 연결되지 않은 채로 Phase 완료 판정됨.

### 누락 레벨 상세

**레벨 1 — 요구사항 수준**
- REQUIREMENTS.md 3.4에 "지능형 라우팅 및 견적 추천"이 명시됨
- 경로 선택 UX(3.4.1~3.4.3)는 Phase 3.3에서 완료 처리됨
- 그러나 "선택된 경로와 정산 엔진의 연계" 항목이 요구사항에 별도 명시되지 않았고, Phase 4 정산 Sprint에서도 연계 작업이 누락됨

**레벨 2 — 구현 수준**
- `zen_route_options` 테이블(segments JSONB)은 구현됐으나 `zen_order_costs` 테이블에 `route_option_id` 컬럼 없음
- `SettlementEngine`이 선택된 경로를 참조하지 않고 단일 운임만 계산
- 구간별(segment) 비용 분해 계산 로직 없음
- 단일 carrier vs 경로별 다른 carrier 처리 로직 없음
- Phase F·G 정산 Task(TASK-035·042·043) 검토 시 Aiden이 route 연계 누락을 감지하지 못함

**레벨 3 — UAT 수준**
- UAT Sprint(TASK-058~064) 작성 시 `zen_route_options` 플로우가 UAT에 반영되지 않음
- UAT 작성 감리 시 요구사항 원문 교차검증 미수행

## 2. 원인 (Why)

- **직접적 원인**: Phase 3.3(라우팅)과 Phase 4(정산)를 독립 Sprint로 분리 진행하면서, 두 Phase 간 데이터 흐름 연계 검증 절차가 존재하지 않았음.
- **근본 원인**:
  - Phase 완료 판정 기준이 해당 Phase 내부 기능 완성 여부에만 집중하고, 후속 Phase와의 연계 정합성 검증을 포함하지 않음.
  - UAT 시나리오 작성 시 구현 코드 기반으로 작성하는 경향으로 인해 요구사항 원문 대조 절차가 생략됨.
- **기여 요소**:
  - 신규 테이블(`zen_route_options`) 생성 후 관련 정산 테이블과의 FK 연계 여부를 별도 감리하는 체크리스트 항목이 부재.
  - Aiden(PM)이 Phase F·G 정산 Task 검토 시 route 참조 여부를 명시적으로 검증하지 않음.

## 3. 조치 (How)

- **이행 계획**:
  - TASK-065 (IMP-070, Type B 구현): `zen_order_costs.route_option_id` FK 추가, `SettlementEngine` 다중 경로 참조 로직 구현, 구간별 비용 분해 계산 및 다중 carrier 처리 로직 추가
  - TASK-066 (Type A UAT 추가): `zen_route_options` 플로우 포함 UAT 시나리오 신규 작성 및 전체 UAT 요구사항 원문 대조 재검증

## 4. 검증 (Verification)

- TASK-065 완료 후: `zen_order_costs.route_option_id` FK 존재 확인, `SettlementEngine` 단위 테스트 다중 경로 케이스 PASS 확인
- TASK-066 완료 후: 추가된 UAT 시나리오가 REQUIREMENTS.md 3.4.1~3.4.3 전 항목을 커버함을 대조표로 증빙

## 5. 예방 (Prevention)

1. **Phase 완료 판정 기준 강화**: Phase 완료 체크리스트에 "후속 Phase(정산·추적)와의 데이터 흐름 연계 검증 필수" 항목 추가
2. **UAT Sprint 착수 전 의무 절차 신설**: UAT Sprint 착수 전 REQUIREMENTS.md 전수 대조를 의무화하고 관련 항목을 체크리스트에 추가
3. **신규 테이블 감리 항목 추가**: 신규 테이블 생성 후 "정산 테이블과 FK 연계 여부" 항목을 코드 리뷰 및 Phase 감리 체크리스트에 포함
