# 1. 문제 개요
Admin Dashboard 내 일부 페이지(`rates`, `settings`, `organizations`)와 관련 컴포넌트에 하드코딩된 다크 테마 배경색(`#0A0A0B`, `#111112`)이 적용되어 프로젝트 디자인 표준인 ZenShell 라이트 모드 (WCAG AA 4.5:1 준수) 정책을 위반하는 현상이 발생했습니다. 

# 2. 원인 분석
개발 단계에서 프리미엄 다크 테마를 개별적으로 하드코딩하여 적용하였으나, 통합 관리 측면에서 프로젝트의 일관된 라이트 모드 컴포넌트 정책(ZenCard)과 충돌하였습니다. 이로 인해 텍스트 시인성 저하 및 접근성 기준(WCAG AA) 미달의 문제를 낳았습니다.

# 3. 조치 내역
다음의 파일들에서 다크 모드 하드코딩을 제거하고 ZenShell 라이트 모드 스타일링으로 일괄 변환하였습니다.

*   `src/app/[locale]/(admin)/rates/page.tsx`
*   `src/app/[locale]/(admin)/settings/page.tsx`
*   `src/app/[locale]/(admin)/organizations/page.tsx`
*   `src/components/admin/RateCardList.tsx`
*   `src/components/admin/RateTierEditor.tsx`
*   `src/components/tracking/AdminTrackingControl.tsx`

**주요 변경 사항:**
- `bg-[#0A0A0B]`, `bg-[#111112]` -> `bg-slate-50`, `bg-white` 
- `text-white/40`, `text-white` 등 -> `text-slate-500`, `text-slate-900` 등 가독성이 보장된 색상으로 전환
- `border-white/5` 등 -> `border-slate-200`
- 버튼 텍스트의 가시성을 확보하기 위한 부분 조정 진행
- `AdminTrackingControl.tsx` 내의 불필요한 `dark:` 클래스 접두사 일괄 제거

# 4. 검증 결과
- `npm run test:regression`을 수행하여 UI 리팩터링이 기존 로직이나 컴포넌트 구조를 훼손하지 않음을 확인. (Exit Code: 0)

# 5. 재발 방지 대책
- 향후 UI 개발 시 독립적인 컬러 코드(Hex 코드 하드코딩) 사용을 지양하고, 반드시 Tailwind CSS `slate` 팔레트 및 `ZenCard`, `ZenInput` 등 제공된 공통 컴포넌트의 기본 속성을 사용하도록 리뷰 과정을 강화합니다.
