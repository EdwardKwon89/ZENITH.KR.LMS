# 1. 문제 개요

Admin Dashboard 내 다수 컴포넌트에서 WCAG AA 기준(4.5:1 대비율) 미달 색상 패턴이 발견되었습니다.  
발견자: Aiden (ZEN_CEO) — E2E-01 실행 중 `/ko/admin/organizations` 직접 발견 (2026-05-01)

## 주요 위반 패턴

| 패턴 | 위반 내용 | 추정 대비율 |
|:---|:---|:---:|
| `via-white/20` in gradient | 다크 배경 가정 그라디언트 (라이트 배경에서 비가시) | N/A |
| `hover:text-blue-300` | 라이트 배경에서 저대비 호버 색상 | ~2.1:1 |
| `bg-amber-500/10 text-amber-500` | 라이트 배경 amber 뱃지 저대비 | ~2.8:1 |
| `bg-slate-900 text-white` (버튼) | ZenShell 비표준 다크 버튼 | OK 대비/비표준 |
| `border-white/20` in forms | 라이트 배경에서 보이지 않는 폼 보더 | N/A |

# 2. 원인 분석

개발 단계에서 일부 컴포넌트에 다크 배경을 가정한 색상 패턴(`via-white/20`, `hover:bg-white/40`) 또는 다크 배경 버튼(`bg-slate-900`)이 적용되었으나, ZenShell 라이트 모드 표준(WCAG AA)과 충돌하였습니다.

# 4. 검증 결과

- `rtk npm run test:regression` → **Exit Code: 0 (PASS)** — 2026-05-01

- 모든 수정 텍스트 색상: `slate-700` 이상 또는 solid 배경 사용으로 WCAG AA 충족

# 5. 재발 방지 대책

1. Admin 컴포넌트 PR 시 `via-white/`, `text-white/[숫자]`, `hover:bg-white/` 패턴 금지 체크리스트 추가
2. `bg-slate-900 text-white` 패턴 사용 시 ZenShell 대안(`bg-blue-600`, `bg-rose-600`, `bg-slate-700`) 우선 사용 권고
3. E2E 시각 검증 단계에서 Admin 화면 스크린샷 기반 색상 표준 준수 확인 포함
