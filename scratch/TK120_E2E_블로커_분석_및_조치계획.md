## 2. 블로커 분석

### Blocker A — RLS Infinite Recursion (CRITICAL · E2E-P6-04)

**원인**:  
`zen_orders` 정책 `"Service providers can view assigned orders"` → `zen_order_services` 조회  
`zen_order_services` 정책 `"order_services_select"` → `zen_orders` 조회  
→ 서로 재귀 호출 → **SQL stack overflow**

**영향**: E2E-P6-04 (역할별 오더 격리) 실행 불가. 또한 운영 DB 영향 가능성.

**Riley 작업물**: `supabase/migrations/20260607020000_fix_rls_infinite_recursion.sql` — SECURITY DEFINER 헬퍼 함수 2개(`is_order_shipper`, `is_order_provider`)로 순환 차단 — **작성 완료 but untracked · 미적용**

**조치**: 
```
supabase migration up    # OR supabase db push (migration 파일 커밋 후)
```

---

### Blocker B — Rate Card Tier Field Name Mismatch (HIGH · E2E-P6-01/02/03)

**원인**: 
- `calculateTransportCost()`가 `tier.rate` 참조
- 실제 DB `zen_rate_cards.tiers[]`는 `unit_price` 필드 사용
- → 계산 결과 전부 `NaN` 또는 `0`

**Riley 작업물**: 
- `service-rates.ts`: `rate`→`unit_price` + 정렬 로직 변경 (desc weight_min, max_weight 검증 제거)
- `p6-db-01.test.ts`, `p6-service-rates.test.ts`, `service-rates.test.ts`: mock 데이터 동기화

**리스크**: 정렬 기준 단순화로 중량구간(weight tier) 오버랩 시 의도치 않은 요율 선택 가능
→ Aiden 검토 필요 (논리 검증)

---

### Blocker C — `fixed_fee` Column Missing (HIGH · E2E-P6-03)

**원인**: `getAvailableServiceRates()`가 `zen_delivery_rates.fixed_fee`를 SELECT → **SQL Error**

**Riley 작업물**: SELECT문과 cost 공식에서 `fixed_fee` 제거 (가격 기대값 변화: `35→25`, `100→70`)

**판정**: `fixed_fee`가 DB에 존재하지 않는 컬럼이면 삭제 정당.  
Riley 측 설명 필요 — 존재해야 하는 컬럼인데 누락된 것인지, 설계상 없는 것인지.

### Blocker D — Auth Guard Whitelist (MEDIUM)

**원인**: `/admin/customs-rates`, `/admin/delivery-rates`가 `proxy.ts` authGuard whitelist에 없음 → 리다이렉트 루프
